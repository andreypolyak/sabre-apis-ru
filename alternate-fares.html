Введение

Branded Fares — инструмент Sabre, предназначенный для поиска, бронирования и оформления билетов по различным тарифным семействам или брендам, указанных перевозчиком. Для каждого бренда определяются условия, такие, как: наличие или отсутствие багажа, возможность обмена и возврата авиабилета, питание на борту, класс обслуживания и т.д. Тарифы разных брендов одного перевозчика могут иметь один класс бронирования, но различаться по коду тарифу.

На данный момент следующие авиакомпании предлагают тарифные семейства, доступные с использованием Branded Fares:

  • Аэрофлот (SU)
  • Уральские Авиалинии (U6)
  • ЮТэйр (UT)
  • Air Berlin (AB)
  • Austrian Airlines (OS)
  • Delta (DL)
  • American Airlines (AA)
  • Etihad (EY)
  • Lufthansa (LH)
  • Swiss (LX)
  • SAS (SK)
  • и другие

Подробнее о Branded Fares можно прочитать на портале Agency eServices. Проверить поддержку Branded Fares определенным перевозчиком можно на сайте Sabre.

Поиск

По умолчанию в результатах поиска сервиса Bargain Finder Max предлагаются варианты перелетов, рассчитанные только по самым дешевым доступным тарифам. Это не всегда удобно для клиента, т.к. зачастую самый дешевый тариф не разрешает провоз багажа, возврат билета и другие дополнительные возможности.

Для получения расчетов стоимости по всем брендам одновременно для всех найденных перелетов необходимо указать у атрибута /OTA_AirLowFareSearchRQ/TravelerInfoSummary/PriceRequestInformation/TPA_Extensions/BrandedFareIndicators@MultipleBrandedFares запроса к сервису Bargain Finder Max значение true.

Бронирование и расчет стоимости

Алгоритм внесения информации о бренде различается в зависимости от того применяется ли один и тот же бренд для всех сегментов бронированирования, применяется несколько разных брендов для разных сегментов или же для некоторых сегментов бренды применяются, а для некоторых нет.

Для получения этой информации необходимо проанализировать ответ сервиса Bargain Finder Max:

  • если все элементы /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/PassengerFare/TPA_Extensions/FareComponents/FareComponent (для самой дешевой комбинации брендов) или /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/TPA_Extensions/AdditionalFares/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/PassengerFare/TPA_Extensions/FareComponents/FareComponent (для дополнительных вариантов) содержат один и тот же код бренда в качестве значения атрибута /@BrandID, то для всего бронирования должен быть выбран один бренд
  • если все указанные выше элементы имеют различный атрибут /@BrandID или некоторые из указанных элементов не имеют атрибута /@BrandID, то при бронировании должен применяться посегментный выбор бренда

Выбор бренда для всего бронирования

При создании бронирования при помощи сервиса EnhancedAirBookRQ, а также расчете стоимости бронирования при помощи сервиса OTA_AirPriceLLSRQ необходимо указать двухбуквенный код бренда в (Brand ID) в элементе /EnhancedAirBookRQ/OTA_AirPriceRQ/PriceRequestInformation/OptionalQualifiers/PricingQualifiers/Brand или /OTA_AirPriceRQ/PriceRequestInformation/OptionalQualifiers/PricingQualifiers/Brand соответственно.

Посегментный выбор бренда

Обратите внимание на то, что многие перевозчики имеют жесткие требования относительно комбинации различных брендов. Не рекомендуется комбинировать их самостоятельно! Результаты поиска Bargain Finder Max будут содержать только те комбинации брендов, которые разрешены перевозчиком.

Для посегментного выбора бренда необходимо определить соответствие между элементами /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/PassengerFare/TPA_Extensions/FareComponents/FareComponent (для самой дешевой комбинации брендов) или /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/TPA_Extensions/AdditionalFares/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/PassengerFare/TPA_Extensions/FareComponents/FareComponent (для дополнительных вариантов) и сегментами. Для этого каждый элемент /FareComponent содержит дочерние элементы /Segment, каждый из которых соответствует одному сегменту в маршруте перелета. Для установления точного соответствия между элементом /Segment и сегментами в путешествии используютсмя атрибуты:

  • /Segment/@LegIndex — номер плеча
  • /Segment/@FlightIndex — номер сегмента в плече

После получения соответствия между брендами и сегментами необходимо в запросе к сервису создания бронирования EnhancedAirBookRQ или к сервису для расчета стоимости OTA_AirPriceLLSRQ указать:

  • последовательности сегментов, имеющих одинаковый бренд или не имеющих бренда в элементах (/EnhancedAirBookRQ)/OTA_AirPriceRQ/PriceRequestInformation/OptionalQualifiers/PricingQualifiers/ItineraryOptions/SegmentSelect со следующими атрибутами:
    • /@Number — номер сегмента начала последовательности
    • /@EndNumber — номер сегмента конца последовательности (опционально)
    • /@RPH — порядковый номер последовательности
  • коды брендов в последовательно расположенных элементах (/EnhancedAirBookRQ)/OTA_AirPriceRQ/PriceRequestInformation/OptionalQualifiers/PricingQualifiers/Brand, где в качестве значения атрибута /@RPH должен быть номер последовательности из предыдущего пункта. Обратите внимание на то, что если для каких-то сегментов код бренда не должен быть использован, то в качестве значения элемента /Brand для этой последовательности должна быть пустая строка

Пример:

Номер сегмента Код бренда
1 AA
2 AA
3 -
4 -
5 BB 
<PricingQualifiers>
  <Brand RPH="1">AA</Brand>
  <Brand RPH="2"/>
  <Brand RPH="3">BB</Brand>
  <ItineraryOptions>
    <SegmentSelect Number="1" EndNumber="2" RPH="1"/>
    <SegmentSelect Number="3" EndNumber="4" RPH="2"/>
    <SegmentSelect Number="5" RPH="3"/>
  </ItineraryOptions>
</PricingQualifiers>

Получение информации о брендах перевозчика

Для получения общей информации по всем доступным брендам перевозчика используется сервис GetMarketingTextRQ. В запросе необходимо указать код перевозчика в атрибутах /GetMarketingTextRQ/BrandsMarketingTextRequest/RequestSource/@requestingCarrierGDS и /GetMarketingTextRQ/BrandsMarketingTextRequest/MarketingTextCriteria/@carrier.

Следует учитывать, что у перевозчика условия брендов могут отличаться в зависимости от направлений перелетов. Для разделения условий по разным направлениям существует системный код программы, который возвращается в атрибуте /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/PassengerFare/TPA_Extensions/FareComponents/FareComponent/@ProgramSystemCode ответа сервиса Bargain Finder Max. В ответе на запрос к сервису GetMarketingTextRQ системный код располагается в элементе /GetMarketingTextRS/BrandsMarketingTextResponse/CarrierBrandsData/BrandProgram/SystemCode.

Авиакомпания S7 Airlines

Авиакомпания S7 Airlines на данный момент не поддерживает Branded Fares. Для экономического класса обслуживания авиакомпания публично файлирует только тарифы группы "Эконом Гибкий" (с багажом и возможностью возврата со штрафом). По запросу агентства авиакомпания может зафайлировать тарифы группы "Эконом Базовый" (без багажа и возможности возврата билета) как приватные для выбранных агентством PCC. Тарифы обеих групп имеют один класс бронирования, но различаются кодом бронирования.

Для бизнес класса авиакомпания файлирует тарифы групп "Бизнес Базовый" и "Бизнес Гибкий" как публичные. Тарифы обеих групп имеют один класс бронирования, но различаются по коду тарифа.

Поиск

Для получения в результатах поиска расчета стоимости перелета по тарифам групп "Эконом Базовый" и "Эконом Гибкий" используется механизм Flexible Fares, при помощи которого можно запросить дополнительный наиболее дешевый публичный тариф. Для этого необходимо добавить элемент /OTA_AirLowFareSearchRQ/TravelPreferences/TPA_Extensions/FlexibleFares/FareParameters/PublicFare с атрибутом /@Ind со значением равным true в запрос к сервису Bargain Finder Max: 

<FlexibleFares>
  <FareParameters>
    <PublicFare Ind="true"/>
  </FareParameters>
</FlexibleFares>

В результатах поиска будет два расчета для одного варианта перелета:

  • Основной по приватному тарифу группы "Эконом Базовый"
  • Альтернативный по публичному тарифу группы "Эконом Гибкий"

Для получения в результатах поиска расчета стоимости перелета по тарифам групп "Бизнес Базовый" и "Бизнес Гибкий" также может быть использован механизм Flexible Fares:

<FlexibleFares>
  <FareParameters> <!-- Бизнес Базовый -->
    <Cabin Type="Business"/>
  </FareParameters>
  <FareParameters> <!-- Бизнес Гибкий -->
    <RefundPenalty Ind="false"/>
    <Cabin Type="Business"/>
  </FareParameters>
</FlexibleFares>

Обратите внимание на то, что возможность получения в результатах поиска расчета по всем доступным брендам для перевозчиков, поддерживающих Branded Fares, не может быть комбинирована с получением в результатах поиска расчета по всем семействам тарифов авиакомпании S7.

Расчет стоимости перелета по всем брендам

Для расчета стоимости перелета по всем доступным брендам можно использовать сервис EnhancedAirBookRQ. Запрос к сервису строится по аналогии с запросом на бронирование (см. Создание бронирований) за исключением:

  • указывается статус сегмента QF (атрибут /EnhancedAirBookRQ/OTA_AirBookRQ/OriginDestinationInformation/FlightSegment/@Status) — специальный статус сегмента, используемый исключительно для расчета стоимости. При бронировании сегментов с таким статусом не отправляется запрос в инвенторную систему перевозчика
  • вместо /EnhancedAirBookRQ/OTA_AirPriceRQ/PriceRequestInformation/@Retain значение true устанавливается у атрибута /EnhancedAirBookRQ/OTA_AirPriceRQ/PriceRequestInformation/@AlternativePricing — запрос альтернативных тарифов вместо сохранения PQ
  • устанавливается значение true у атрибута /EnhancedAirBookRQ/OTA_AirPriceRQ/PriceRequestInformation/OptionalQualifiers/PricingQualifiers/AlternativePricing/Overrides/MatchNoMatch/@Ind — запрос альтернативных тарифов, как в запрошенном классе бронирования, так и в других классах бронирования
  • у атрибутов /EnhancedAirBookRQ/PreProcessing/@IgnoreBefore и /EnhancedAirBookRQ/PostProcessing/@IgnoreAfter устанавливается значение true — игнорирование текущего бронирования до и после выполнения запроса

В результате работы сервиса создаются фиктивные сегменты со статусом QF, для которых производится расчет стоимости по альтернативным тарифам (аналог терминальной команды WPA), после чего бронирование игнорируется. Ответ на такой запрос будет содержать список доступных для бронирования тарифов с указанием стоимости, класса бронирования, нормы провоза багажа и другой информации.

Бронирование и расчет стоимости

Бронирование и расчет стоимости для тарифов групп "Эконом Базовый" и "Бизнес Базовый" производится по умолчанию и не требует дополнительных изменений в запросе на бронирование.

Для бронирования и расчета стоимости перелетов по тарифам, относящимся к группе "Эконом Гибкий", необходимо в запросе к сервису EnhancedAirBookRQ (или OTA_AirPriceLLSRQ) указать значение true у атрибута (/EnhancedAirBookRQ)/OTA_AirPriceRQ/PriceRequestInformation/OptionalQualifiers/PricingQualifiers/FareOptions/@Public.

Для бронирования и расчета стоимости перелетов по тарифам, относящимся к группе "Бизнес Гибкий", необходимо в запросе к сервису EnhancedAirBookRQ (или OTA_AirPriceLLSRQ) указать значение true у атрибута (/EnhancedAirBookRQ)/OTA_AirPriceRQ/PriceRequestInformation/OptionalQualifiers/PricingQualifiers/SpecificPenalty/Refundable/@Any.
