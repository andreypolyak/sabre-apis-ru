Введение

Обмен билетов при работе через Sabre APIs осуществляется при помощи дополнительного продукта Sabre Automated Exchanges, который автоматизирует процесс поиска, расчета стоимости и оформления билетов в обмен. Для подключения продукта, пожалуйста, обратитесь к вашему куратору в Sabre. Продукт позволяет обменять любой билет, в котором был применен тариф (или тарифы), содержащий 31-ю категорию правил.

В данном руководстве предложен алгоритм выполнения одинакового обмена всех билетов, которые имеются в бронировании для всех пассажиров. В случае, если необходимо обменять только некоторые билеты или необходимо выполнить разные обмены билетов (т.е. новые билеты должны быть оформлены для разных перелетов), то сначала необходимо выполнить деление бронирования (см. Деление бронирований).

В данном разделе описан процесс обмена билетов. Процесс поиска вариантов перелета для обмена описан в разделе Поиск вариантов перелетов для обмена.

Переход в другой PCC (ContextChangeLLSRQ)

Обмен билетов необходимо производить в том PCC, где они были изначально оформлены. В случае, если билеты не были оформлены в iPCC, то необходимо перейти в другой PCC, отправив запрос к сервису ContextChangeLLSRQ.

Обратите внимание на то, что для перехода в другой PCC требуется наличие Branch Access между ним и iPCC, в котором была создана сессия. Подробнее о Branch Access см. разделе Конфигурация Sabre.

В качестве значения атрибута /ContextChangeRQ/ChangeAAA/@PseudoCityCode в запросе указывается PCC, в котором будет произведен обмен.

Пример запроса
Копировать 
<ContextChangeRQ ReturnHostCommand="true" Version="2.0.3" xmlns="http://webservices.sabre.com/sabreXML/2011/10">
  <ChangeAAA PseudoCityCode="2FRH"/>
</ContextChangeRQ>

Чтение бронирования (TravelItineraryReadRQ)

Для чтения бронирования используется сервис TravelItineraryReadRQ.

В качестве значения атрибута /TravelItineraryReadRQ/UniqueID/@ID в запросе указывается код бронирования (PNR Record Locator).

Пример запроса
Копировать

Удаление старых сегментов (OTA_CancelLLSRQ)

Для удаления старых сегментов, которые необходимо поменять, используется сервис OTA_CancelLLSRQ.

В запросе к сервису необходимо указать какие именно сегменты должны быть отменены:

  • <Segment Type="entire"/> — все сегменты
  • <Segment Type="air"/> — все авиасегменты
  • <Segment Number="1"/> — первый сегмент
  • <Segment Number="2" EndNumber="4"/> — все сегменты со второго по четвертый включительно

Вместе с сегментами будут отменены привязанные к ним дополнительные услуги.

Номера сегментов могут быть получены в ответе на запрос TravelItineraryReadRQ в TravelItinerary/ItineraryInfo/ReservationItems/Item/@RPH.

<OTA_CancelRQ ReturnHostCommand="true" Version="2.0.2" xmlns="http://webservices.sabre.com/sabreXML/2011/10">

Бронирование новых сегментов (EnhancedAirBookRQ)

Для бронирования новых сегментов используется сервис EnhancedAirBookRQ.

В запросе необходимо указать:

  • /EnhancedAirBookRQ/OTA_AirBookRQ/HaltOnStatus/@Code — коды статусов сегментов, при получении которых должно быть остановлено выполнение запроса
  • /EnhancedAirBookRQ/OTA_AirBookRQ/OriginDestinationInformation/FlightSegment — бронируемые сегменты (подробнее см. Создание бронирований в 2 шага)
  • /EnhancedAirBookRQ/OTA_AirBookRQ/RedisplayReservation — проверка статуса сегментов после бронирования:
    • /@NumAttempts — количество проверок
    • /@WaitInterval — интервал между проверками в миллисекундах

  • /EnhancedAirBookRQ/@HaltOnError — остановка выполнения подзапросов в случае получения ошибки

Отправка SSR и сохранение бронирования (PassengerDetailsRQ)

После бронирования новых сегментов необходимо повторно отправить все SSR, которые должны быть привязаны к новым сегментам, а также сохранить бронирование. Для этого используется сервис PassengerDetailsRQ.

Обратите внимание на то, что повторно требуется отправить только те SSR, которые "привязываются" к сегментам (например DOCS, DOCO, DOCA, INFT, CHLD и другие). SSR, которые не привязываются к сегментам (например FOID), а также OSI сообщения повторно отправлять не требуется.

В запросе необходимо указать:

  • /PassengerDetailsRQ/SpecialReqDetails/SpecialServiceRQ/SpecialServiceInfo/AdvancePassenger — SSR типа DOCS, DOCO и DOCA (подробнее см. Создание бронирований в 2 шага)
  • /PassengerDetailsRQ/SpecialReqDetails/SpecialServiceRQ/SpecialServiceInfo/Service — другие SSR (подробнее см. Создание бронирований в 2 шага)
  • /PassengerDetailsRQ/PostProcessing/EndTransactionRQ/EndTransaction/@Ind — признак сохранения бронирования (значение true)
  • /PassengerDetailsRQ/PostProcessing/EndTransactionRQ/Source/@ReceivedFrom — значения поля Received From. Используется для идентификации инициатора изменений в истории бронирования
  • /PassengerDetailsRQ/@HaltOnError — остановка выполнения подзапросов в случае получения ошибки

Сохранение маски расчета стоимости обмена (AutomatedExchangesLLSRQ)

Для сохранения маски расчета стоимости обмена (PQR) необходимо повторно отправить запрос к сервису AutomatedExchangesLLSRQ.

В запросе необходимо указать:

  • /AutomatedExchangesRQ/ExchangeConfirmation/@PQR_Number — номер создаваемой PQR. Обратите внимание на то, что PQ и PQR имеют сквозную систему нумерации и в качестве значения данного атрибута необходимо указать первый незанятый номер PQ или PQR
  • /AutomatedExchangesRQ/ExchangeConfirmation/@DoNotIssueEMDForChangeFee — признак необходимости оформления EMD во время обмена. Значение true устанавливается тогда, когда EMD оформлять не требуется
  • /AutomatedExchangesRQ/ExchangeConfirmation/OptionalQualifiers/FOP_Qualifiers/BasicFOP — форма оплаты. Возможные варианты:
    • наличная форма оплаты — значение CA у атрибута /@Type
    • безналичиная форма оплаты — значение CA у атрибута /@Type
    • кредитная карта — требуется заполнение следующих атрибутов:
      • /@Code — код платежной системы
      • /@ExpireDate — год и месяц истечения действия карты
      • /@ManualApprovalCode — код авторизации платежа (в случае наличия)
      • /@Number — номер карты

Пример запроса
Копировать 
<AutomatedExchangesRQ ReturnHostCommand="true" Version="2.6.1" xmlns="http://webservices.sabre.com/sabreXML/2011/10">
  <ExchangeConfirmation DoNotIssueEMDForChangeFee="true" PQR_Number="4">
    <OptionalQualifiers>
      <FOP_Qualifiers>
        <BasicFOP Type="CA"/>
      </FOP_Qualifiers>
    </OptionalQualifiers>
  </ExchangeConfirmation>
</AutomatedExchangesRQ>
