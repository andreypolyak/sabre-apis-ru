Введение

В Sabre APIs существует два сервиса получения текста правил тарифов:

  • OTA_AirRulesLLSRQ — получение текста правил тарифов по заданным параметрам тарифа
  • RulesFromPriceLLSRQ — получение текста правил тарифов после выполнения расчета стоимости в бронировании

Ниже представлены рекомендации по использованию этих сервисов.

Получение текста правил тарифов по заданным параметрам тарифа (OTA_AirRulesLLSRQ)

Для получения текста правил тарифов по заданным параметрам тарифа используется сервис OTA_AirRulesLLSRQ. Сервис позволяет запрашивать текст правил тарифов без необходимости создания бронирования.

Составление запроса

В запросе необходимо указать:

  • /OTA_AirRulesRQ/OriginDestinationInformation/FlightSegment/OriginLocation/@LocationCode — пункт отправления тарифа
  • /OTA_AirRulesRQ/OriginDestinationInformation/FlightSegment/DestinationLocation/@LocationCode — пункт прибытия тарифа

Обратите внимание на то, что пункты отправления и прибытия тарифа могут не совпадать с пунктами отправления и прибытия сегментов. О том, как получить эти и другие данные для формирования запроса см. ниже.

  • /OTA_AirRulesRQ/OriginDestinationInformation/FlightSegment/@DepartureDateTime — дата вылета
  • /OTA_AirRulesRQ/RuleReqInfo/FareBasis/@Code — код тарифа
  • /OTA_AirRulesRQ/OriginDestinationInformation/FlightSegment/MarketingCarrier/@Code — код маркетингового перевозчика

Для получения правил приватного тарифа также может потребоваться указание Account Code (/OTA_AirRulesRQ/OptionalQualifiers/PricingQualifiers/Account/Code) или Corporate ID (/OTA_AirRulesRQ/OptionalQualifiers/PricingQualifiers/Corporate/ID).

В случае необходимости получения только определенных категорий правил тарифа необходимо указать номера категорий пассажиров в последовательно расположенных элементах /OTA_AirRulesRQ/RuleReqInfo/Category.

Выбор тарифа из списка дубликатов

В некоторых случаях перевозчики могут иметь несколько тарифов с одинаковым кодом, пунктом отправления, пунктом прибытия и датой вылета. Для выбора одного из предложенных тарифов необходимо повторно отправить запрос к сервису OTA_AirRulesLLSRQ, указав номер тарифа из списка в качестве значения атрибута /OTA_AirRulesRQ/RuleReqInfo/@RPH.

Копировать 
<OTA_AirRulesRQ ReturnHostCommand="true" Version="2.2.1" xmlns="http://webservices.sabre.com/sabreXML/2011/10">
  <RuleReqInfo RPH="1">

  </RuleReqInfo>
</OTA_AirRulesRQ>

Алгоритм получения данных из ответа сервиса Bargain Finder Max (OTA ответ)

Ответ сервиса Bargain Finder Max (а также Bargain Finder Max Alternate Date и Bargain Finder Max Alternate Airport Shop) содержит информацию обо всех тарифах для всех сегментов в последовательно расположенных элементах /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/FareBasisCodes/FareBasisCode. Каждый элемент будет соответствовать одному сегменту в маршруте (элемент /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItinerary/OriginDestinationOptions/OriginDestinationOption/FlightSegment).

В некоторых случаях один тариф может быть применен к нескольким сегментам. В этом случае ему будут соответствовать несколько элементов /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/FareBasisCodes/FareBasisCode, однако у второго и последующих элементов будут отсутствовать атрибуты /@FareComponentBeginAirport, /@FareComponentEndAirport, /@FareComponentDirectionality и /@FareComponentVendorCode. Запрашивать правила такого тарифа в этом случае нужно только один раз.

Ниже представлено соответствие между элементами в запросе к сервису OTA_AirRulesLLSRQ и элементами в ответе сервиса Bargain Finder Max:

Пункт отправления тарифа

  • Элемент в запросе к сервису OTA_AirRulesLLSRQ: /OTA_AirRulesRQ/OriginDestinationInformation/FlightSegment/OriginLocation/@LocationCode
  • Элемент в ответе сервиса Bargain Finder Max: /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/FareBasisCodes/FareBasisCode/@FareComponentBeginAirport (если атрибут /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/FareBasisCodes/FareBasisCode/@FareComponentDirectionality имеет значение FROM) или /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/FareBasisCodes/FareBasisCode/@FareComponentEndAirport (если атрибут /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/FareBasisCodes/FareBasisCode/@FareComponentDirectionality имеет значение TO)

Пункт прибытия тарифа

  • Элемент в запросе к сервису OTA_AirRulesLLSRQ: /OTA_AirRulesRQ/OriginDestinationInformation/FlightSegment/DestinationLocation/@LocationCode
  • Элемент в ответе сервиса Bargain Finder Max: /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/FareBasisCodes/FareBasisCode/@FareComponentEndAirport (если атрибут /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/FareBasisCodes/FareBasisCode/@FareComponentDirectionality имеет значение FROM) или /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/FareBasisCodes/FareBasisCode/@FareComponentBeginAirport (если атрибут /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/FareBasisCodes/FareBasisCode/@FareComponentDirectionality имеет значение TO)

Дата вылета

  • Элемент в запросе к сервису OTA_AirRulesLLSRQ: /OTA_AirRulesRQ/OriginDestinationInformation/FlightSegment/@DepartureDateTime
  • Элемент в ответе сервиса Bargain Finder Max: /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItinerary/OriginDestinationOptions/OriginDestinationOption/FlightSegment/@DepartureDateTime (дата вылета соответствующего сегмента)

Код тарифа

  • Элемент в запросе к сервису OTA_AirRulesLLSRQ: /OTA_AirRulesRQ/RuleReqInfo/FareBasis/@Code
  • Элемент в ответе сервиса Bargain Finder Max: /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/FareBasisCodes/FareBasisCode

Код маркетингового перевозчика

  • Элемент в запросе к сервису OTA_AirRulesLLSRQ: /OTA_AirRulesRQ/OriginDestinationInformation/FlightSegment/MarketingCarrier/@Code
  • Элемент в ответе сервиса Bargain Finder Max: /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/FareBasisCodes/FareBasisCode/@GovCarrier

Получение текста правил тарифов после выполнения расчета стоимости в бронировании (RulesFromPriceLLSRQ)

Для получения текста правил тарифов по заданным параметрам тарифа OTA_AirRulesLLSRQ. Сервис позволяет запрашивать текст правил тарифов без необходимости создания бронирования.

Составление запроса

В запросе нет обязательных элементов, кроме корневого.

Однако, в случае, если расчет стоимости был выполнен для пассажиров разных категорий (например, для взрослого и ребенка) или был выполнен расчет с использованием нескольких тарифов, потребуется указать:

  • /RulesFromPriceRQ/RuleReqInfo/Passenger/@Code — код категории пассажира
  • /RulesFromPriceRQ/RuleReqInfo/SegmentSelect/@Number — номер сегмента в бронировании, для которого требуется получить текст правил тарифа

В случае необходимости получения только определенных категорий правил тарифа необходимо указать номера категорий пассажиров в последовательно расположенных элементах /RulesFromPriceRQ/RuleReqInfo/Category.
