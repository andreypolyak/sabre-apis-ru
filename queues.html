Введение

Очереди в Sabre — инструмент для контроля и учета бронирований, а также уведомления об изменении их состояния. Каждая очередь обладает уникальным номером от 0 до 511, при этом очереди с номерами от 0 до 49 являются системными, а остальные могут быть созданы пользователями самостоятельно. Бронирования попадают в системные очереди автоматически при возникновении определенных событий. Например, бронирование автоматически попадает в системную очередь 7 при удалении сегментов перевозчиком. При помещении бронирования в очередь используется PIC (Prefactory Instruction Code) — код причины постановки бронирования в очередь.

Получение списка очередей

Для получения списка очередей используется сервис QueueCountLLSRQ.

По умолчанию запрашивается список очередей для текущего PCC, однако в атрибуте /QueueCountRQ/QueueInfo/QueueIdentifier/@PseudoCityCode можно задать другой PCC (требуется наличие Branch Access).

Пример запроса

<QueueCountRQ ReturnHostCommand="true" Version="2.1.0" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <QueueInfo>
    <QueueIdentifier PseudoCityCode="9LSC"/>
  </QueueInfo>
</QueueCountRQ>

Получение списка бронирований в очереди

Для получения списка бронирований в очереди используется сервис QueueAccessLLSRQ.

В запросе должны быть указаны:

  • /QueueAccessRQ/QueueIdentifier/@Number — номер очереди
  • /QueueAccessRQ/QueueIdentifier/@PseudoCityCode — PCC, в котором находится очередь
  • /QueueAccessRQ/QueueIdentifier/List/@Ind — индикатор получения списка бронирований в очереди (значение true)

Пример запроса

<QueueAccessRQ ReturnHostCommand="true" Version="2.0.6" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <QueueIdentifier Number="20" PseudoCityCode="9LSC">
    <List Ind="true"/>
  </QueueIdentifier>
</QueueAccessRQ>

Вход в очередь

Для того, чтобы выполнить определенное действие с бронированиями, находящимися в очереди, необходимо предварительно совершить вход в очередь. Для этого используется сервис QueueAccessLLSRQ.

В запросе должны быть указаны:

  • /QueueAccessRQ/QueueIdentifier/@Number — номер очереди
  • /QueueAccessRQ/QueueIdentifier/@PseudoCityCode — PCC, в котором находится очередь

В ответе на запрос будет содержаться последнее бронирование, добавленное в эту очередь.

Пример запроса

  <QueueIdentifier Number="20" PseudoCityCode="9LSC"/>
</QueueAccessRQ>

Управление бронированиями в очереди

Для управления бронированиями в очереди используется сервис QueueAccessLLSRQ.

Механизм работы с очередями в Sabre устроен таким образом, что после входа в очередь всегда должно быть открыто какое-либо бронирование. Текущее бронирование можно редактировать (см. Чтение и редактирование бронирований), а также выполнить одно из следующих действий:

  • Сохранить изменения и оставить в очереди — значение E в /QueueAccessRQ/Navigation/@Action
  • Игнорировать изменения и оставить в очереди — значение I в /QueueAccessRQ/Navigation/@Action
  • Сохранить изменения и удалить из очереди — значение E в /QueueAccessRQ/Navigation/Direction/@Action
  • Игнорировать изменения и удалить из очереди — значение I в /QueueAccessRQ/Navigation/Direction/@Action

Выход из очереди

Для выхода из очереди используется сервис QueueAccessLLSRQ.

При выходе из бронирования можно сохранить изменения (значение QXE /QueueAccessRQ/Navigation/@Action) в текущем бронировании или игнорировать их (значение QXI).

Пример запроса

<QueueAccessRQ ReturnHostCommand="true" Version="2.0.6" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <Navigation Action="QXI"/>
</QueueAccessRQ>

Пример ответа

<QueueAccessRS Version="2.0.6" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:stl="http://services.sabre.com/STL/v01" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <stl:ApplicationResults status="Complete">
    <stl:Success timeStamp="2016-05-01T14:27:31-05:00">
      <stl:SystemSpecificResults>
        <stl:HostCommand LNIATA="469BEF">QXI</stl:HostCommand>
      </stl:SystemSpecificResults>
    </stl:Success>
  </stl:ApplicationResults>
  <Paragraph>
    <Text>OFF 9LSC QUEUE 020</Text>
  </Paragraph>
</QueueAccessRS>

Постановка бронирования в очередь

Бронирование можно поместить в очередь самостоятельно одним из двух способов:

  • При создании бронирования в подзапросе QueuePlaceRQ сервиса PassengerDetailsRQ (см. Создание бронирований)
  • В любой момент времени, используя сервис QueuePlaceLLSRQ

В запросе к сервису QueuePlaceLLSRQ необходимо указать следующие параметры:

  • /QueuePlaceRQ/QueueInfo/QueueIdentifier/@Number — номер очереди
  • /QueuePlaceRQ/QueueInfo/QueueIdentifier/@PrefatoryInstructionCode — PIC, код причины постановки в очередь
  • /QueuePlaceRQ/QueueInfo/QueueIdentifier/@PseudoCityCode — PCC
  • /QueuePlaceRQ/QueueInfo/UniqueID/@ID — код бронирования (PNR Record Locator)

Пример запроса

<QueuePlaceRQ ReturnHostCommand="true" Version="2.0.4" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <QueueInfo>
    <QueueIdentifier Number="400" PrefatoryInstructionCode="11" PseudoCityCode="9LSC"/>
    <UniqueID ID="DLWEKR"/>
  </QueueInfo>
</QueuePlaceRQ>

Пример ответа

<QueuePlaceRS Version="2.0.4" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:stl="http://services.sabre.com/STL/v01" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <stl:ApplicationResults status="Complete">
    <stl:Success timeStamp="2016-05-01T15:08:42-05:00">
      <stl:SystemSpecificResults>
        <stl:HostCommand LNIATA="21C5F0">QP/9LSC400/11*DLWEKR</stl:HostCommand>
      </stl:SystemSpecificResults>
    </stl:Success>
  </stl:ApplicationResults>
  <QueueInfo DateTime="05-01T15:08">
    <QueueIdentifier Number="0400" PseudoCityCode="9LSC"/>
    <UniqueID ID="DLWEKR"/>
  </QueueInfo>
  <Text>QUEUE PLACEMENT COMPLETED</Text>
</QueuePlaceRS>

Обработка изменений в расписании

При обработке очередей номер 5 (изменение расписания вылета до 17 дней) и 6 (изменение расписания вылета более 17 дней) требуется подтверждение сегментов со статусом TK агентством. Для этого необходимо отправить команду .HKALL при помощи сервиса SabreCommandLLSRQ:

Пример запроса

<SabreCommandLLSRQ Version="1.8.1" xmlns="http://webservices.sabre.com/sabreXML/2003/07" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <Request Output="SCREEN">
    <HostCommand>.HKALL</HostCommand>
  </Request>
</SabreCommandLLSRQ>

Предварительно рекомендуется отправить команду VCT* для проверки минимального стыковочного времени в бронировании (в случае наличия двух или более сегментов):

Пример запроса

<SabreCommandLLSRQ Version="1.8.1" xmlns="http://webservices.sabre.com/sabreXML/2003/07" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <Request Output="SCREEN">
    <HostCommand>VCT*</HostCommand>
  </Request>
</SabreCommandLLSRQ>

Ответ должен содержать MINIMUM CONNECT TIME EDIT VALID FOR ALL CONNECTIONS:

Пример ответа

<SabreCommandLLSRS AltLangID="en-us" EchoToken="String" PrimaryLangID="en-us" SequenceNmbr="1" Target="Production" TimeStamp="2016-05-06T16:04:27" Version="1.8.1" xmlns="http://webservices.sabre.com/sabreXML/2003/07">
  <Response>MINIMUM CONNECT TIME EDIT VALID FOR ALL CONNECTIONS</Response>
</SabreCommandLLSRS>

После подтверждения сегментов необходимо сохранить бронирование, отправив команду EW при помощи сервиса SabreCommandLLSRQ:

Пример запроса

<SabreCommandLLSRQ Version="1.8.1" xmlns="http://webservices.sabre.com/sabreXML/2003/07" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <Request Output="SCREEN">
    <HostCommand>EW</HostCommand>
  </Request>
</SabreCommandLLSRQ>
