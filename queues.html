Введение

Очереди в Sabre — инструмент для контроля и учета бронирований, а также уведомления об изменении их состояния. Каждая очередь обладает уникальным номером от 0 до 511, при этом очереди с номерами от 0 до 49 являются системными, а остальные могут быть созданы пользователями самостоятельно. Бронирования попадают в системные очереди автоматически при возникновении определенных событий. Например, бронирование автоматически попадает в системную очередь 7 при удалении сегментов перевозчиком.

Алгоритм обработки очередей

Существует два основных способа обработки очередей:

  • stateful обработка очередей, эмулирующая работу агентов с очередями в терминале (со входом в очереди):
    1. получение списка очередей
    2. вход в очередь
    3. обработка бронирований в очереди
    4. навигация по очереди
  • stateless обработка очередей (без входа в очередь)
    1. получение списка очередей
    2. получение списка бронирований в очереди
    3. обход всех бронирований с целью обработки и удаления из очереди

В данном руководстве представлены оба алгоритма работы с очередями.

Получение списка очередей (QueueCountLLSRQ)

Для получения списка очередей используется сервис QueueCountLLSRQ.

По умолчанию запрашивается список очередей для текущего PCC, однако в атрибуте /QueueCountRQ/QueueInfo/QueueIdentifier/@PseudoCityCode можно задать другой PCC (требуется наличие Branch Access).

Пример запроса

<QueueCountRQ ReturnHostCommand="true" Version="2.1.0" xmlns="http://webservices.sabre.com/sabreXML/2011/10">
  <QueueInfo>
    <QueueIdentifier PseudoCityCode="9LSC"/>
  </QueueInfo>
</QueueCountRQ>

Вход в очередь (QueueAccessLLSRQ)

Для того, чтобы выполнить определенное действие с бронированиями, находящимися в очереди, необходимо предварительно совершить вход в очередь. Для этого используется сервис QueueAccessLLSRQ.

В запросе должны быть указаны:

  • /QueueAccessRQ/QueueIdentifier/@Number — номер очереди
  • /QueueAccessRQ/QueueIdentifier/@PseudoCityCode — PCC, в котором находится очередь

В ответе на запрос будет содержаться последнее бронирование, добавленное в эту очередь.

Пример запроса

<QueueAccessRQ ReturnHostCommand="true" Version="2.0.7" xmlns="http://webservices.sabre.com/sabreXML/2011/10">
  <QueueIdentifier Number="20" PseudoCityCode="9LSC"/>
</QueueAccessRQ>

Получение бронирования в структурированном виде (TravelItineraryReadRQ)

Редактирование бронирования

Текущее открытое бронирование можно изменить. См. Редактирование бронирований, Оформление билетов и EMD, Отмена бронирований, Войдирование билетов и EMD и др.

Обратите внимание на то, что если по завершению выполнения изменений требуется сохранить или игнорировать бронирование, то вместо сервисов EndTransactionLLSRQ и IgnoreTransactionLLSRQ необходимо вызывать сервис QueueAccessLLSRQ с соответствующими параметрами (см. ниже). Однако, если при изменении бронирования были внесены изменения в сегментах (например, при отмене бронирования или изменении статусов сегментов) для сохранения бронирования предварительно нужно обновить поле Received From, отправив запрос к сервису EndTransactionLLSRQ:

Пример запроса

<EndTransactionRQ ReturnHostCommand="true" Version="2.0.6" xmlns="http://webservices.sabre.com/sabreXML/2011/10">
  <Source ReceivedFrom="SWS"/>
</EndTransactionRQ>

Принятие изменений в расписании

См. ниже.

Сохранение изменений, сохранение бронирования в очереди и показ следующего бронирования (QueueAccessLLSRQ)

Пример запроса

<QueueAccessRQ ReturnHostCommand="true" Version="2.0.7" xmlns="http://webservices.sabre.com/sabreXML/2011/10">
  <Navigation Action="EL"/>
</QueueAccessRQ>

Сохранение изменения, сохранение бронирования в очереди и показ его заново (QueueAccessLLSRQ)

Пример запроса

<QueueAccessRQ ReturnHostCommand="true" Version="2.0.7" xmlns="http://webservices.sabre.com/sabreXML/2011/10">
  <Navigation Action="ER"/>
</QueueAccessRQ>

Сохранение изменения, удаление бронирования из очереди и показ следующего бронирования (QueueAccessLLSRQ)

Пример запроса

<QueueAccessRQ ReturnHostCommand="true" Version="2.0.7" xmlns="http://webservices.sabre.com/sabreXML/2011/10">

Редактирование бронирования

Текущее открытое бронирование можно изменить. См. Редактирование бронирований, Оформление билетов и EMD, Отмена бронирований, Войдирование билетов и EMD и др.

Удаление бронирования из очереди (QueueAccessLLSRQ)

Для удаления бронирования из очереди используется сервис QueueAccessLLSRQ.

В запросе необходимо указать:

  • /QueueAccessRQ/Navigation/@Action — тип операции (значение QR)
  • /QueueAccessRQ/QueueIdentifier/@PseudoCityCode — PCC
  • /QueueAccessRQ/QueueIdentifier/@Number — номер очереди

Пример запроса

<QueueAccessRQ ReturnHostCommand="true" Version="2.0.7" xmlns="http://webservices.sabre.com/sabreXML/2011/10">
  <Navigation Action="QR"/>
  <QueueIdentifier PseudoCityCode="9LSC" Number="7"/>
</QueueAccessRQ>

Пример ответа

<QueueAccessRS Version="2.0.7" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:stl="http://services.sabre.com/STL/v01" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <stl:ApplicationResults status="Complete">
    <stl:Success timeStamp="2016-11-03T16:09:15-05:00">
      <stl:SystemSpecificResults>
        <stl:HostCommand LNIATA="38D40B">QR/9LSC7</stl:HostCommand>
      </stl:SystemSpecificResults>
    </stl:Success>
  </stl:ApplicationResults>
</QueueAccessRS>

Сохранение бронирования (EndTransactionLLSRQ)

Для сохранения изменений в бронировании необходимо отправить запрос к сервису EndTransactionLLSRQ. В результате выполнения запроса все изменения в бронировании, выполненные в текущей сессии, будут сохранены, а бронирование закрыто.

В запросе нужно указать код ReceivedFrom, он будет отображаться в истории бронирования.

Постановка бронирования в очередь (QueuePlaceLLSRQ)

Бронирование можно поместить в очередь самостоятельно одним из двух способов:

  • При создании бронирования в подзапросе QueuePlaceRQ сервиса PassengerDetailsRQ (см. Создание бронирований)
  • В любой момент времени, используя сервис QueuePlaceLLSRQ

В запросе к сервису QueuePlaceLLSRQ необходимо указать следующие атрибуты:

  • /QueuePlaceRQ/QueueInfo/QueueIdentifier/@Number — номер очереди
  • /QueuePlaceRQ/QueueInfo/QueueIdentifier/@PrefatoryInstructionCode — PIC, код причины постановки в очередь
  • /QueuePlaceRQ/QueueInfo/QueueIdentifier/@PseudoCityCode — PCC
  • /QueuePlaceRQ/QueueInfo/UniqueID/@ID — код бронирования (PNR Record Locator)

Пример запроса

<QueuePlaceRQ ReturnHostCommand="true" Version="2.0.5" xmlns="http://webservices.sabre.com/sabreXML/2011/10">
  <QueueInfo>
    <QueueIdentifier Number="400" PrefatoryInstructionCode="11" PseudoCityCode="9LSC"/>
    <UniqueID ID="DLWEKR"/>
  </QueueInfo>
</QueuePlaceRQ>

Пример ответа

<QueuePlaceRS Version="2.0.5" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:stl="http://services.sabre.com/STL/v01" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">

Обработка изменений в расписании

При обработке очередей с номером 5 (изменение расписания вылета до 17 дней) и 6 (изменение расписания вылета более 17 дней) требуется подтверждение агентством сегментов со статусом TK. Для этого необходимо отправить команду .HKALL при помощи сервиса SabreCommandLLSRQ:

Пример запроса

<SabreCommandLLSRQ Version="1.8.1" xmlns="http://webservices.sabre.com/sabreXML/2003/07">
  <Request Output="SCREEN">
    <HostCommand>.HKALL</HostCommand>
  </Request>
</SabreCommandLLSRQ>

Предварительно рекомендуется отправить команду VCT* для проверки минимального стыковочного времени в бронировании (в случае наличия двух или более сегментов):

Пример запроса

<SabreCommandLLSRQ Version="1.8.1" xmlns="http://webservices.sabre.com/sabreXML/2003/07">
  <Request Output="SCREEN">
    <HostCommand>VCT*</HostCommand>
  </Request>
</SabreCommandLLSRQ>

Ответ должен содержать MINIMUM CONNECT TIME EDIT VALID FOR ALL CONNECTIONS:

Пример ответа

<SabreCommandLLSRS AltLangID="en-us" EchoToken="String" PrimaryLangID="en-us" SequenceNmbr="1" Target="Production" TimeStamp="2016-05-06T16:04:27" Version="1.8.1" xmlns="http://webservices.sabre.com/sabreXML/2003/07">
  <Response>MINIMUM CONNECT TIME EDIT VALID FOR ALL CONNECTIONS</Response>
</SabreCommandLLSRS>

После подтверждения сегментов необходимо сохранить бронирование, отправив команду EW при помощи сервиса SabreCommandLLSRQ:

Пример запроса

<SabreCommandLLSRQ Version="1.8.1" xmlns="http://webservices.sabre.com/sabreXML/2003/07">
  <Request Output="SCREEN">
    <HostCommand>EW</HostCommand>
  </Request>
</SabreCommandLLSRQ>

Обработка бронирований с прошедшим тайм-лимитом или статусами сегментов HX

При обработке очередей с номерами 7 (перевозчик изменил статус сегмента или сегментов на HX), 9 и 10 (наступил тайм-лимит) возникает необходимость удаления сегментов (снятие мест). Для этого используется сервис OTA_CancelLLSRQ (см. Отмена бронирований).

После удаления сегментов необходимо сохранить изменения в бронировании (см. выше).
