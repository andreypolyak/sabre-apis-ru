Деление бронирований

Оглавление:

  • Введение
  • Алгоритм деления бронирования
  • Чтение бронирования (TravelItineraryReadRQ)
  • Выбор пассажиров для переноса в другое бронирование (TravelItineraryDivideLLSRQ)
  • Добавление поля Received From в новое бронирование (EndTransactionLLSRQ)
  • Сохранение нового бронирования (TravelItineraryDivideLLSRQ)
  • Сохранение оригинального бронирования (EndTransactionLLSRQ)

Введение

В некоторых случаях, например, при отказе от бронирования одного из пассажиров в PNR или при изменении условий перевозки для части пассажиров, необходимо разделить созданное бронирование. При выделении пассажиров в новый PNR автоматически перемещаются все ассоциированные с пассажиром элементы, а также номера всех оформленных билетов.

Чтение бронирования (TravelItineraryReadRQ)

Сервис TravelItineraryReadRQ позволяет открыть бронирование в текущей сессии и прочесть его содержимое.

В запросе необходимо указать код бронирования (PNR Record Locator).

Пример запроса

    <SubjectAreas>
      <SubjectArea>FULL</SubjectArea>
    </SubjectAreas>
  </MessagingDetails>
  <UniqueID ID="ODFAFF"/>
</TravelItineraryReadRQ>

Выбор пассажиров для переноса в другое бронирование (TravelItineraryDivideLLSRQ)

Для того чтобы выбрать пассажиров в текущем бронировании для переноса в другое бронирование используется сервис TravelItineraryDivideLLSRQ.

В запросе необходимо указать один или несколько номеров пассажиров, которые нужно перенести в отдельное бронирование:

  • <PersonName NameNumber="2.1"/> — пассажир с номером 2.1
  • <PersonName EndNameNumber="3.1" NameNumber="1.1"/> — пассажиры 1.1, 2.1, 3.1
  • <PersonName EndNameNumber="2.1" NameNumber="1.1"/> <PersonName NameNumber="4.1"/> — пассажиры 1.1, 2.1, 4.1

Пример запроса

<TravelItineraryDivideRQ ReturnHostCommand="true" Version="2.0.2" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <Divide>
    <CustomerInfo>
      <PersonName NameNumber="2.1"/>
    </CustomerInfo>
  </Divide>
</TravelItineraryDivideRQ>

Пример ответа

<TravelItineraryDivideRS Version="2.0.2" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:stl="http://services.sabre.com/STL/v01" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">

Добавление поля Received From в новое бронирование (EndTransactionLLSRQ)

Перед тем как сохранить новое бронирование необходимо добавить поле Received From, для этого требуется отправить запрос к сервису EndTransactionLLSRQ.

В запросе нужно указать код ReceivedFrom, он будет отображаться в истории бронирования.

Пример запроса

<EndTransactionRQ ReturnHostCommand="true" Version="2.0.5" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <Source ReceivedFrom="SWS"/>
</EndTransactionRQ>

Пример ответа

<EndTransactionRS Version="2.0.5" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:stl="http://services.sabre.com/STL/v01" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <stl:ApplicationResults status="Complete">
    <stl:Success timeStamp="2016-05-23T04:50:18-05:00">
      <stl:SystemSpecificResults>
        <stl:HostCommand LNIATA="36B4FC">6SWS</stl:HostCommand>
      </stl:SystemSpecificResults>
