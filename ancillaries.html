Алгоритм бронирования дополнительных услуг

Варианты алгоритма:

Разница между двумя алгоритмами состоит в том, что в случае получения списка дополнительных услуг в отдельном запросе, необходимо игнорировать бронирование и заново его открыть т.к. за то время, что клиент выбирает дополнительные услуги, бронирование могло измениться. В случае получения списка дополнительных услуг в результатах поиска, пользователь может выбрать дополнительные услуги до момента начала процесса бронирования дополнительных услуг.

Схема

Чтение бронирования (TravelItineraryReadRQ)

Для чтения бронирования используется сервис TravelItineraryReadRQ.

В качестве значения атрибута /TravelItineraryReadRQ/UniqueID/@ID в запросе указывается код бронирования (PNR Record Locator).

Обратите внимание на то, что:

  • многие перевозчики присылают код-локаторы сегментов (/TravelItineraryReadRS/TravelItinerary/ItineraryInfo/ReservationItems/Item/FlightSegment/SupplierRef/@ID) не мгновенно после создания бронирования, а с некоторой задержкой
  • некоторые перевозчики после создания бронирования отправляют сообщения (/TravelItineraryReadRS/TravelItinerary/SpecialServiceInfo) с правилами оформления билета (тайм-лимитом) после создания бронирования

В связи с этим рекомендуется делать некоторую задержку между созданием бронирования и отправкой запроса на чтение бронирования в том случае, если используется процесс при котором клиент выбирает дополнительные услуги из ответа на запрос к сервису Bargain Finder Max (получение списка дополнительных услуг в результатах поиска). Если клиент выбирает дополнительные услуги из ответа на запрос к сервису OTA_AirPriceLLSRQ, то дополнительной задержки добавлять не требуется.

Получение списка дополнительных услуг для бронирования (OTA_AirPriceLLSRQ)

Для получения списка дополнительных услуг требуется отправить запрос к сервису OTA_AirPriceLLSRQ. В запросе необходимо указать значение true у атрибута /OTA_AirPriceRQ/PriceRequestInformation/OptionalQualifiers/MiscQualifiers/AirExtras/@Ind, а также указать категорию пассажира, для которого необходимо получить список дополнительных услуг, в качестве значения атрибута /OTA_AirPriceRQ/PriceRequestInformation/OptionalQualifiers/PricingQualifiers/PassengerType/@Code (обратите внимание на то, что атрибут /@Quantity должен быть пустым). Дополнительные услуги не могут быть забронированы для младенца без места.

<OTA_AirPriceRQ ReturnHostCommand="true" Version="2.16.0" xmlns="http://webservices.sabre.com/sabreXML/2011/10">
  <PriceRequestInformation>
    <OptionalQualifiers>
      <MiscQualifiers>
        <AirExtras Ind="true"/>
      </MiscQualifiers>
      <PricingQualifiers> 
        <PassengerType Code="ADT" Quantity=""/> 
      </PricingQualifiers> 
    </OptionalQualifiers>
  </PriceRequestInformation>
</OTA_AirPriceRQ>

Оформление EMD для дополнительных услуг

Для платных дополнительных услуг обязательно требуется оформление EMD. Обратите внимание на то, что:

  • EMD должны быть оформлены в бронировании после билетов
  • для оформления EMD статус дополнительной услуги в бронировании должен иметь значение HD (атрибут /TravelItineraryReadRS/TravelItinerary/ItineraryInfo/ReservationItems/Item/Ancillaries/AncillaryService/@ActionCode ответа на запрос к сервису TravelItineraryReadRQ)

Процесс оформления EMD подробно описан в разделе Оформление билетов и EMD.
