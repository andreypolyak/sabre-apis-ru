Чтение бронирования (TravelItineraryReadRQ)

Сервис TravelItineraryReadRQ позволяет открыть бронирование в текущей сессии и прочесть его содержимое.

В качестве параметра UniqueID/@ID в запросе указывается код бронирования (PNR Record Locator).

Пример запроса

<TravelItineraryReadRQ Version="3.6.0" xmlns="http://services.sabre.com/res/tir/v3_6">
  <MessagingDetails>
    <SubjectAreas>
      <SubjectArea>FULL</SubjectArea>
    </SubjectAreas>
  </MessagingDetails>
  <UniqueID ID="NPBYOK"/>
</TravelItineraryReadRQ>

Изменение бронирования (TravelItineraryModifyInfoLLSRQ)

Используя сервис TravelItineraryModifyInfoLLSRQ можно изменить значения для следующих элементов бронирования:

  • имя пассажира
  • тип пассажира
  • контактная информация (телефон, email)
  • номер карты часто летающего пассажира
  • адрес агентства
  • значение тайм лимита

Сохранение бронирования (EndTransactionLLSRQ)

Для того чтобы сохранить изменения в бронировании необходимо отправить запрос к сервису EndTransactionLLSRQ. В результате все изменения в бронировании, выполненные в текущей сессии, будут сохранены, а бронирование закрыто.

В запросе нужно указать код ReceivedFrom, он будет отображаться в истории бронирования.

Пример запроса

<EndTransactionRQ ReturnHostCommand="true" Version="2.0.5" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <EndTransaction Ind="true"/>
  <Source ReceivedFrom="SWS"/>
</EndTransactionRQ>

Пример ответа

<EndTransactionRS Version="2.0.5" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:stl="http://services.sabre.com/STL/v01" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <stl:ApplicationResults status="Complete">
    <stl:Success timeStamp="2016-01-31T11:20:36-06:00">
      <stl:SystemSpecificResults>
        <stl:HostCommand LNIATA="222222">6SWS§E</stl:HostCommand>
      </stl:SystemSpecificResults>
    </stl:Success>
  </stl:ApplicationResults>
  <ItineraryRef ID="NPBYOK">
    <Source CreateDateTime="2016-01-31T11:20"/>
  </ItineraryRef>
  <Text>OK 1120 NPBYOK</Text>
</EndTransactionRS>

Игнорирование бронирования (IgnoreTransactionLLSRQ)

Иногда бывает необходимость игнорировать все изменения в бронировании, выполненные в текущей сессии. Для этого необходимо использовать сервис IgnoreTransactionLLSRQ.

Пример запроса

<IgnoreTransactionRQ xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" ReturnHostCommand="true" Version="2.0.0"/>

Пример ответа

<IgnoreTransactionRS Version="2.0.0" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:stl="http://services.sabre.com/STL/v01" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <stl:ApplicationResults status="Complete">
    <stl:Success timeStamp="2016-02-01T02:43:23-06:00">
      <stl:SystemSpecificResults>
        <stl:HostCommand LNIATA="222222">I</stl:HostCommand>
      </stl:SystemSpecificResults>
    </stl:Success>
  </stl:ApplicationResults>
</IgnoreTransactionRS>
