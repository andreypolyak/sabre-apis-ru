Введение

У каждого PCC существует ряд настроек, которые называются TJR (Travel Journal Record). Настройки TJR определяют различные аспекты работы системы Sabre и применимы, как для работы через Sabre APIs, так и для работы через терминал. Указанные ниже обязательные и необязательные настройки рекомендуется устанавливать во всех используемых PCC (включая iPCC).

Настройки PCC могут быть изменены супервайзером агентства после прохождения соответствующего тренинга (терминальные команды указаны ниже) или по заявке, направленной в Sabre Helpdesk.

Список настроек PCC представлен в алфавитном порядке.

Проверка настроек

На этой странице можно проверить состояние указанных настроек в вашем PCC. Для этого необходимо отправить указанные ниже команды в терминале Sabre Red Workspace или отправить запрос к сервису SabreCommandLLSRQ (см. Отправка терминальных команд).

Вставьте в поле ниже результаты выполнения команды W/TA*[ваш PCC], например W/TA*9LSC.

Обратите внимание, что в терминале Sabre Red Workspace есть ограничение на количество строк в ответе. Для того, чтобы пролистать ответ на команду до конца, необходимо последовательно отправлять команду MD.

Вставьте в поле ниже результаты выполнения команды W/K*TJR.

Обратите внимание, что перед выполнением команды необходимо перейти в тот PCC, настройки, которого вы хотите получить. Это можно сделать при помощи терминальной команды AAA[ваш PCC], например AAA9LSC.

Результаты проверки настроек:

Alert to Prevent Duplicate Ticketing (Предупреждение об оформлении дубликатов билетов)

✓ Настройку рекомендуется включить

Описание

Описание в Format Finder.

Настройка Alert to Prevent Duplicate Ticketing вызывает предупреждение в случае попытки повторного оформления билета для тех пассажиров и тех сегментов, для которых в бронировании уже существует билет.

Если настройка включена

При попытке оформить билет для тех пассажиров и тех сегментов, для которых в бронировании уже существует билет, в ответ на запрос AirTicketLLSRQ будет вызвано предупреждение:

Проверка наличия дубликата билета не производится.

Allow Ticket CAT 35 IT/BT PQ Fare (Оформление билетов по сохраненным в PQ приватным тарифам без перерасчета)

✓ Настройку рекомендуется включить

Описание

Описание в Format Finder.

Настройка Allow Ticket CAT 35 IT/BT PQ Fare разрешает оформление билетов по приватным тарифам при включенной настройке Ticket from Stored Fare (см. ниже).

Если настройка включена

Оформление билетов по приватным тарифам при включенной настройке Ticket from Stored Fare разрешено.

Если настройка отключена

Оформление билетов по приватным тарифам при включенной настройке Ticket from Stored Fare не разрешено.

Включение и отключение

  • W/C35IT¥* — проверить состояние настройки
  • W/C35IT¥ON — включить настройку
  • W/C35IT¥OFF — отключить настройку

Auto Segment Arrangement (Автоматическое упорядочивание сегментов в бронированиях)

✓ Настройку рекомендуется включить

Описание

Описание в Format Finder.

Настройка Auto Segment Arrangement автоматически выстраивает сегменты по порядку (т.е. в хронологическом порядке) при добавлении сегментов в бронирование.

Обратите внимание на то, что данная настройка не отменяет необходимости добавления наземных сегментов в случае наличия пропусков в бронировании (т.е. в тех случаях, когда город отправления следующего сегмента не равен городу прибытия предыдущего сегмента). Для автоматического добавления наземных сегментов можно использовать элемент /PassengerDetailsRQ/PostProcessing/ARUNK_RQ в запросе к сервису PassengerDetailsRQ (см. Создание бронирований).

Если настройка включена

При добавлении (бронировании) сегментов они будут выстроены в бронировании в правильном порядке.

Если настройка отключена

При добавлении (бронировании) сегментов они будут выстроены в том порядке, котором были добавлены. При попытке сохранения бронирования при помощи запроса к сервису EndTransactionLLSRQ будет получена ошибка:

Пример ответа
Копировать 
<EndTransactionRS Version="2.0.6" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:stl="http://services.sabre.com/STL/v01" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <stl:ApplicationResults status="NotProcessed">
    <stl:Error timeStamp="2016-12-09T08:17:00-06:00" type="BusinessLogic">
      <stl:SystemSpecificResults>
        <stl:HostCommand LNIATA="764168">6SWS§E</stl:HostCommand>
        <stl:Message>SEGMENTS NOT IN DATE ORDER - VERIFY AND REENTER</stl:Message>
        <stl:Message>VERIFY ORDER OF ITINERARY SEGMENTS - MODIFY OR END TRANSACTION</stl:Message>
        <stl:ShortText>ERR.SWS.HOST.ERROR_IN_RESPONSE</stl:ShortText>
      </stl:SystemSpecificResults>
    </stl:Error>
  </stl:ApplicationResults>
</EndTransactionRS>

Включение и отключение

  • W/TA*[PCC] затем MD/ARRANGE — проверить состояние настройки
  • W/AZA¥ON — включить настройку
  • W/AZA¥OFF — отключить настройку

Automatically End Transaction and Redisplay the PNR at Ticketing (Автоматическое сохранение бронирований и повторное их открытие при оформлении билетов)

✗ Настройку рекомендуется отключить

Описание

Описание в Format Finder.

Настройка Automatically End Transaction at Ticketing автоматически сохраняет бронирование после выполнения запроса на оформление, войдирование и возврат билетов или EMD, после чего заново читает сохраненное бронирование.

Обратите внимание на то, что все рекомендации в разделах Оформление билетов и EMD, Войдирование билетов и EMD и других составлены с тем расчетом, что данная настройка отключена. В случае ее включения необходимо исключить из рекомендованных процессов запрос на чтение бронирования после выполнения запроса на оформление, войдирование или возврат билетов или EMD.

Данная настройка совместима со скриптом для расчета НДС, используемом в Sabre Red Workspace.

Если настройка включена

После выполнения запроса на оформление, войдирование или возврат билетов или EMD будет произведено сохранение бронирование и его повторное чтение. После выполнения запроса в текущей сессии или вкладке в Sabre Red Workspace бронирование будет открыто.

Если настройка отключена

После выполнения запроса на оформление, войдирование или возврат билетов или EMD запрошенная транзакция будет выполнена (т.е., например, билет оформлен или возвращен), однако информация об этой транзакции не будет присутствовать в бронировании до тех пор, пока оно не будет сохранено.

Включение и отключение

  • W/AUTOER¥* — проверить состояние настройки
  • W/AUTOER¥ON — включить настройку
  • W/AUTOER¥OFF — отключить настройку

Automatically End Transaction at Ticketing (Автоматическое сохранение бронирований при оформлении билетов)

✓ Настройку рекомендуется включить

Описание

Описание в Format Finder.

Настройка Automatically End Transaction at Ticketing автоматически сохраняет бронирование после выполнения запроса на оформление, войдирование и возврат билетов или EMD. Данная настройка повышает надежность оформления билетов.

Обратите внимание на то, что все рекомендации в разделах Оформление билетов и EMD, Войдирование билетов и EMD и других составлены с тем расчетом, что данная настройка включена.

Также обратите внимание на то, что данная настройка не совместима со скриптом для расчета НДС, используемом в Sabre Red Workspace. В случае использования этого скрипта рекомендуется использовать настройку Automatically End Transaction and Redisplay the PNR at Ticketing (см. ниже).

Если настройка включена

После выполнения запроса на оформление, войдирование или возврат билетов или EMD будет произведено сохранение бронирование. После выполнения запроса в текущей сессии или вкладке в Sabre Red Workspace бронирование не будет открыто.

Если настройка отключена

После выполнения запроса на оформление, войдирование или возврат билетов или EMD запрошенная транзакция будет выполнена (т.е., например, билет оформлен или возвращен), однако информация об этой транзакции не будет присутствовать в бронировании до тех пор, пока оно не будет сохранено.

Включение и отключение

  • W/AUTOEND¥* — проверить состояние настройки
  • W/AUTOEND¥ON — включить настройку
  • W/AUTOEND¥OFF — отключить настройку

Enhanced PQ (Улучшенные PQ)

✓ Настройку рекомендуется включить

Описание

Описание в Format Finder.

Настройка Enhanced PQ улучшает многие возможности PQ, такие как:

  • использование PQ при автоматическом и ручном ценообразовании
  • хранение истории изменений всех PQ
  • возможность редактирования PQ
  • возможность подключения настройки Ticket from Stored PQ Fare (см. ниже)
  • и другие

Если настройка включена

Создаются улучшенные PQ (см. возможности выше).

Если настройка отключена

Улучшенные PQ не создаются.

Включение и отключение

  • W/PQPLUS¥* — проверить состояние настройки
  • W/PQPLUS¥ON — включить настройку
  • W/PQPLUS¥OFF — отключить настройку

Minimum Connect Time Edit (Проверка соответствия пересадок минимальному допустимому времени)

✓ Настройку рекомендуется включить

Описание

Описание в Format Finder.

Настройка Minimum Connect Time Edit включает проверку на соответствие всех пересадок в бронировании минимальному допустимому времени пересадки.

Если настройка включена

При попытке сохранения бронирования, в котором длительность одной или нескольких пересадок меньше допустимого минимального времени пересадки будет показано предупреждение:

Пример ответа
Копировать 
<PassengerDetailsRS xmlns="http://services.sabre.com/sp/pd/v3_3">
  <ns2:ApplicationResults status="NotProcessed" xmlns:ns2="http://services.sabre.com/STL_Payload/v02_01">
    <ns2:Error timeStamp="2016-12-15T14:25:35.512-06:00" type="BusinessLogic">
      <ns2:SystemSpecificResults>
        <ns2:Message code="ERR.SWS.HOST.ERROR_IN_RESPONSE">MIN CONNX TIME SEG 01 AT SVO 1.00</ns2:Message>
      </ns2:SystemSpecificResults>
    </ns2:Error>
  </ns2:ApplicationResults>
</PassengerDetailsRS>

Для того чтобы сохранить такое бронирование, невзирая на предупреждение, необходимо отправить повторный запрос на сохранение бронирования (сервис EndTransactionLLSRQ). В противном случае бронирование нужно игнорировать (IgnoreTransactionLLSRQ).

Если настройка отключена

Проверка соответствия пересадок в бронировании минимальному допустимому времени пересадок не производится.

Включение и отключение

  • W/K*TJR, в ответе должна быть строка MCT — проверить состояние настройки
  • W/KMCT¥ON — включить настройку
  • W/KMCT¥OFF — отключить настройку

Price Retention Ticketing Alert (Предупреждение об изменении стоимости билетов)

✗ Настройку рекомендуется отключить

Описание

Описание в Format Finder.

В Sabre по умолчанию (в случае, если не включена настройка Ticket from Stored Fare) каждый раз при попытке оформить билеты производится перерасчет стоимости оформляемых билетов. Настройка Price Retention Ticketing Alert вызывает проверку соответствия стоимости билета (или билетов), сохраненных в PQ, стоимости билета или билетов, которые будут оформлены в результате выполнения запроса. В случае, если стоимость билетов различается, будет вызвано предупреждение.

Обратите внимание на то, что по умолчанию в бронировании сохраняется 1 PQ на всех пассажиров одной категории, и, при попытке оформить билет по данному PQ для меньшего числа пассажиров, будет вызвано предупреждение.

Например, в случае наличия в бронировании 3 взрослых пассажиров и 2 детей, в бронировании будет по умолчанию создано 2 PQ: 1 для взрослых пассажиров и 1 для детей. При оформлении билетов по этим PQ по-пассажирно (т.е. для 1 пассажира в 1 запросе на оформление билета) будет вызвано предупреждение.

Если настройка включена

При попытке оформить билет стоимость которого отличается от стоимости сохраненной в PQ в ответе на запрос AirTicketLLSRQ будет вызвано предупреждение:

Пример ответа
Копировать 
<AirTicketRS Version="2.9.0" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:stl="http://services.sabre.com/STL/v01" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <stl:ApplicationResults status="NotProcessed">
    <stl:Error timeStamp="2016-12-09T09:31:15-06:00" type="BusinessLogic">
      <stl:SystemSpecificResults>
        <stl:HostCommand LNIATA="764168">W¥PQ1N1.1¥FCK¥K1</stl:HostCommand>
        <stl:Message>VERIFY TKT TTL  RUB     23050 - PQ TTL  RUB     46100</stl:Message>
        <stl:Message>TICKET?  ENTER Y OR N&lt;.&gt;</stl:Message>
        <stl:ShortText>ERR.SWS.HOST.ERROR_IN_RESPONSE</stl:ShortText>
      </stl:SystemSpecificResults>
    </stl:Error>
  </stl:ApplicationResults>
</AirTicketRS>

Для того чтобы все-таки оформить билет необходимо отправить следующий запрос к сервису SabreCommandLLSRQ (см. Отправка терминальных команд):

Пример запроса
Копировать 
<SabreCommandLLSRQ Version="1.8.1" xmlns="http://webservices.sabre.com/sabreXML/2003/07">
  <Request Output="SCREEN">
    <HostCommand><![CDATA[VE<Y>]]></HostCommand>
  </Request>
</SabreCommandLLSRQ>

Для того чтобы отказаться от оформления билетов необходимо отправить следующий запрос к сервису SabreCommandLLSRQ:

Пример запроса
Копировать 
<SabreCommandLLSRQ Version="1.8.1" xmlns="http://webservices.sabre.com/sabreXML/2003/07">
  <Request Output="SCREEN">
    <HostCommand><![CDATA[VE<N>]]></HostCommand>
  </Request>
</SabreCommandLLSRQ>

Если настройка отключена

Сравнение соответствия стоимости оформляемых билетов не производится.

Включение и отключение

  • W/PQT¥* — проверить состояние настройки
  • W/PQT¥ON — включить настройку
  • W/PQT¥OFF — отключить настройку

PSPT Warning Message (Предупреждение об отсутствии паспортов в бронировании)

✗ Настройку рекомендуется отключить

Описание

Описание в Format Finder.

Настройка PSPT Warning Message вызывает предупреждение об отсутствии паспорта при попытке оформить билет.

Обратите внимание на то, что предупреждение будет показано всегда в том случае, если в бронировании присутствует младенец, т.к. документ (паспорт) будет "привязан" к сопровождающему младенца взрослому пассажиру.

Если настройка включена

При попытке оформить билет для пассажира, у которого в бронировании отсутствует паспорт, будет выдано предупреждение:

  <stl:ApplicationResults status="NotProcessed">
    <stl:Error timeStamp="2016-12-09T08:27:57-06:00" type="BusinessLogic">
      <stl:SystemSpecificResults>
        <stl:HostCommand LNIATA="764168">W¥PQ1¥FCK¥K1</stl:HostCommand>
        <stl:Message>WARNING - NO PASSPORT AND/OR DOB DATA FOR SELECTED PAX</stl:Message>
        <stl:Message>CONTINUE TICKETING Y OR N &gt;»&lt;.&gt;</stl:Message>
        <stl:ShortText>ERR.SWS.HOST.ERROR_IN_RESPONSE</stl:ShortText>

</AirTicketRS>

  </Request>
</SabreCommandLLSRQ>

Для того чтобы отказаться от оформления билетов необходимо отправить следующий запрос к сервису SabreCommandLLSRQ:

Пример запроса

  <Request Output="SCREEN">
    <HostCommand><![CDATA[WA<N>]]></HostCommand>
  </Request>
</SabreCommandLLSRQ>

Если настройка отключена

Проверка наличия в бронировании паспорта не производится.

Включение и отключение

  • W/TKM¥* — проверить состояние настройки
  • W/TKM¥ON — включить настройку
  • W/TKM¥OFF — отключить настройку

Store Passenger Type in PNR (Хранение категорий пассажиров в бронированиях)

✓ Настройку рекомендуется включить

Описание

Описание в Format Finder.

Настройка Store Passenger Type in PNR позволяет включить сохранение категорий пассажиров в бронировании (категории пассажира указываются в качестве значения атрибута /PassengerDetailsRQ/TravelItineraryAddInfoRQ/CustomerInfo/PersonName/@PassengerType в запросе к сервису для добавления информации о пассажирах PassengerDetailsRQ).

Если настройка включена

При расчете стоимости для всех пассажиров будут применены те категории пассажиров, что указаны в бронировании.

При расчете стоимости по умолчанию для всех пассажиров будет применена категория пассажира ADT (взрослый). В запросе на расчет стоимости можно указать другие категории пассажиров.

Включение и отключение

  • W/PT¥* — проверить состояние настройки
  • W/PT¥ON — включить настройку
  • W/PT¥OFF — отключить настройку

Ticket from Stored Fare (Оформление билетов по сохраненным PQ без перерасчета)

✓ Настройку рекомендуется включить

Описание

Описание в Format Finder.

В Sabre по умолчанию каждый раз при попытке оформить билеты производится перерасчет стоимости оформляемых билетов. Настройка Ticket from Stored Fare отключает перерасчет стоимости бронирования в случае, если PQ был создан в течение текущего дня (используется локальное время PCC). Гарантия Sabre при этом сохраняется.

При включении данной настройки также рекомендуется включить настройку Allow Ticket CAT 35 IT/BT PQ Fare (см. выше) для того, чтобы иметь возможность оформлять билеты по приватным тарифам.

Если настройка включена

При оформлении билетов не производится перерасчет стоимости PQ. В случае, если при оформлении билетов, использовался PQ, созданный не в текущий день, а ранее, будет показано предупреждение:

Пример ответа
Копировать 
<AirTicketRS Version="2.9.0" xmlns="http://webservices.sabre.com/sabreXML/2011/10" xmlns:stl="http://services.sabre.com/STL/v01" xmlns:xs="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <stl:ApplicationResults status="NotProcessed">
    <stl:Error timeStamp="2016-12-09T10:12:21-06:00" type="BusinessLogic">
      <stl:SystemSpecificResults>
        <stl:HostCommand LNIATA="764168">W¥PQ1¥FCK¥K1</stl:HostCommand>
        <stl:Message>UNABLE TO TICKET STORED FARE - PQ EXPIRED</stl:Message>
        <stl:Message>ENTER R-REPRICE/TKT OR Q-QUIT OR O-OVERRIDE/TKT&lt;.&gt;</stl:Message>
        <stl:ShortText>ERR.SWS.HOST.ERROR_IN_RESPONSE</stl:ShortText>
      </stl:SystemSpecificResults>
    </stl:Error>
  </stl:ApplicationResults>
</AirTicketRS>

Для того чтобы выполнить перерасчет стоимости и оформить билет по актуальной стоимости необходимо отправить следующий запрос к сервису SabreCommandLLSRQ (см. Отправка терминальных команд):

Пример запроса
Копировать 
<SabreCommandLLSRQ Version="1.8.1" xmlns="http://webservices.sabre.com/sabreXML/2003/07">
  <Request Output="SCREEN">
    <HostCommand><![CDATA[UN<R>]]></HostCommand>
  </Request>
</SabreCommandLLSRQ>

Для того чтобы оформить билет по сохраненному PQ без перерасчета (в этом случае гарантия Sabre не сохранится) необходимо отправить следующий запрос к сервису SabreCommandLLSRQ:

Пример запроса
Копировать 
<SabreCommandLLSRQ Version="1.8.1" xmlns="http://webservices.sabre.com/sabreXML/2003/07">
  <Request Output="SCREEN">
    <HostCommand><![CDATA[UN<O>]]></HostCommand>
  </Request>
</SabreCommandLLSRQ>

Для того чтобы отказать от оформления билетов необходимо отправить следующий запрос к сервису SabreCommandLLSRQ:

Пример запроса
Копировать 
<SabreCommandLLSRQ Version="1.8.1" xmlns="http://webservices.sabre.com/sabreXML/2003/07">
  <Request Output="SCREEN">
    <HostCommand><![CDATA[UN<Q>]]></HostCommand>
  </Request>
</SabreCommandLLSRQ>

Если настройка отключена

При оформлении билетов будет выполнен перерасчет стоимости.

Включение и отключение

  • W/TKTPQ¥* — проверить состояние настройки
  • для включения или отключения настройки необходимо обратиться к вашему куратору в Sabre
