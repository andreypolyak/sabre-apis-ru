Поиск по гибким направлениям

Поиск перелетов по гибким направлениям

Оглавление:

  • Введение
  • Тип запроса
  • Маршрут
  • Пример

Введение

Для поиска перелетов по гибким направлениямпо заданному маршруту с гибкими датами используется сервис Bargain Finder Max Alternate Airport Shop (BargainFinderMax_ASRQ).

Обратите внимание на то, что использование этого сервиса требует активации для каждого iPCC. Пожалуйста, обратитесь к вашему куратору в Sabre для уточнения деталей.

Схема запроса и ответа сервиса BargainFinderMax_ASRQ, а также правила составления запросов совпадают с таковыми для сервиса BargainFinderMaxRQ (см. Поиск перелетов по заданным датам) за исключением перечисленных ниже особенностей.

Тип запроса

Тип запроса (элемент /OTA_AirLowFareSearchRQ/TPA_Extensions/IntelliSellTransaction/RequestType) указывать не требуется.

Маршрут

Сервис доступен только для маршрутов с перелетом в одну сторону (One Way) или туда-обратно (Round Trip).

Сервис позволяет указать в запросе до 20 различных аэропортов в качестве пункта отправления и до 20 различных аэропортов в качестве пункта назначения для поиска самых дешевых перелетов между ними. Общее число комбинаций аэропортов в пунктах отправления и назначения не может превышать 32. Например, вы можете указать 4 аэропорта в пункте отправления и 8 аэропортов в пункте назначения (4×8=32≤32), но не можете указать 6 аэропортов в пункте отправления и 6 аэропортов в пункте назначения (6×6=36>32).

При поиске Round-Trip перелетов, состоящего из двух плеч (туда и обратно), аэропорты отправления для первого плеча должны совпадать с аэропортами назначения для второго плеча, аэропорты назначения для первого плеча должны совпадать с аэропортами отправления для второго плеча.

Пример корректного поиска:

  1. SVO, DME, VKO, ZIA → CDG, ORY, NCE, LYS, TLS
  2. CDG, ORY, NCE, LYS, TLS → SVO, DME, VKO, ZIA

Пример некорректного поиска (аэропорты отправления второго плеча не совпадают с аэропортами назначения первого плеча):

  1. SVO, DME, VKO, ZIA → CDG, ORY, NCE, LYS, TLS
  2. FCO, MXP, VCE → SVO, DME, VKO, ZIA

Обратите внимание на то, что в качестве пунктов отправления и назначения можно указывать только коды аэропортов, но не городов.

Код первого аэропорта отправления для каждого плеча должен быть указан в качестве значения атрибута /OTA_AirLowFareSearchRQ/OriginDestinationInformation/OriginLocation/@LocationCode. Коды остальных аэропортов отправления должны быть указаны в качестве значений атрибутов /@LocationCode последовательно расположенных элементов /OTA_AirLowFareSearchRQ/OriginDestinationInformation/TPA_Extensions/SisterOriginLocation.

Код первого аэропорта назначения для каждого плеча должен быть указан в качестве значения атрибута /OTA_AirLowFareSearchRQ/OriginDestinationInformation/DestinationLocation/@LocationCode. Коды остальных аэропортов отправления должны быть указаны в качестве значений атрибутов /@LocationCode последовательно расположенных элементов /OTA_AirLowFareSearchRQ/OriginDestinationInformation/TPA_Extensions/SisterDestinationLocation.

Ответ будет содержать как минимум по одному самому дешевому варианту перелета для каждой указанной пары городов, остальные варианты будут выбрану по принципу минимальной стоимости вне зависимости от направления.
