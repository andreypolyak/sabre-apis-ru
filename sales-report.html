Отчет о продажах (TKT_TravelAgencyReportsRQ)

Для получения отчета о продажах используется сервис TKT_TravelAgencyReportsRQ.

Сервис позволяет получить следующий список документов (в скобках указан их код):

  • оформленных билетов (TKT)
  • возвратов билетов (RFND)
  • оформленных EMD (EMD)
  • возвратов EMD (RFEMD)

По умолчанию сервис позволяет получить список билетов, EMD и возвратов билетов и EMD для последней доступной даты для всех возможных стоков (BSP, прямые стоки перевозчиков, ТКП) в iPCC. Для этого необходимо отправить следующий запрос:

Пример запроса
Копировать 
<DailySalesSummaryRQ version="1.2.2" xmlns="http://www.sabre.com/ns/Ticketing/AsrServices/1.0">
  <Header/>
  <SelectionCriteria/>
</DailySalesSummaryRQ>

Дополнительно можно указать:

  • /DailySalesSummaryRQ/SelectionCriteria/PseudoCityCode — PCC, для которого будет сформирован отчет. Обратите внимание на то, что при формировании отчета для другого PCC будет произведена проверка наличия доступа (Branch Access) между iPCC и указанным PCC
  • /DailySalesSummaryRQ/SelectionCriteria/Airline/AirlineCode — код авиакомпании из двух букв (например, SU для Аэрофлота)
  • /DailySalesSummaryRQ/SelectionCriteria/Airline/AirlineNumber — код авиакомпании из трех цифр (например, 555 для Аэрофлота)
  • /DailySalesSummaryRQ/SelectionCriteria/ReportDate — дата, за которую необходимо получить отчет о продажах. Указанная дата не может быть ранее, чем 1 сентября 2016 года (2016-09-01)
  • /DailySalesSummaryRQ/SelectionCriteria/DocumentType — тип документа (см. коды выше)
  • /DailySalesSummaryRQ/SelectionCriteria/DocumentNumber — номер документа (билета или EMD, 13 цифр)

Пример запроса
Копировать 
<DailySalesSummaryRQ version="1.2.2" xmlns="http://www.sabre.com/ns/Ticketing/AsrServices/1.0">
  <Header/>
  <SelectionCriteria>
    <PseudoCityCode>02AA</PseudoCityCode>
    <ReportDate>2016-09-02</ReportDate>
    <DocumentType>TKT</DocumentType>
  </SelectionCriteria>
</DailySalesSummaryRQ>

Отчет о неиспользованных билетах

Переход в другой PCC (ContextChangeLLSRQ)

Для формирования отчета в другом PCC необходимо перейти в него, отправив запрос к сервису ContextChangeLLSRQ. Запрос не потребуется, если отчет необходимо сформировать для билетов оформленных iPCC.

Обратите внимание на то, что для перехода в другой PCC требуется наличие Branch Access между ним и iPCC, в котором была создана сессия. Подробнее о Branch Access см. в разделе Конфигурация Sabre.

В качестве значения атрибута /ContextChangeRQ/ChangeAAA/@PseudoCityCode в запросе указывается PCC, в котором должно быть создано бронирование.

Пример запроса
Копировать 
<ContextChangeRQ ReturnHostCommand="true" Version="2.0.3" xmlns="http://webservices.sabre.com/sabreXML/2011/10">
  <ChangeAAA PseudoCityCode="2FRH"/>
</ContextChangeRQ>

Пример ответа

Формирование отчета (UnusedeTicketReportLLSRQ)

Для формирования отчетов о неиспользованных билетах используется сервис UnusedeTicketReportLLSRQ. Неиспользованным считается билет, у которого прошло 5 дней с момента вылета у последнего сегмента и в билете имеются купоны со статусом OPEN.

Обратите внимание на то, что для работы этого сервиса требуется включение настройки Unused Electronic Ticket Report. Подробнее см. Настройки PCC.

В запросе дополнительно можно указать:

  • /Unused_eTicketReportRQ/eTicketReport/@CustomerIdentifier — DK код клиента
  • /Unused_eTicketReportRQ/eTicketReport/@StartDate — дата добавления билета в отчет или дата начала временного промежутка (добавления билета в отчет)
  • /Unused_eTicketReportRQ/eTicketReport/@EndDate — дата окончания временного промежутка (добавления билета в отчет)
  • /Unused_eTicketReportRQ/eTicketReport/VendorPrefs/Airline — код перевозчика

Пример запроса
Копировать 
<UnusedeTicketReportRQ ReturnHostCommand="true" Version="2.0.0" xmlns="http://webservices.sabre.com/sabreXML/2011/10">
  <eTicketReport EndDate="08-10" StartDate="08-01">
    <VendorPrefs>
      <Airline Code="SU"/>
    </VendorPrefs>
  </eTicketReport>
</UnusedeTicketReportRQ>

Пример ответа
