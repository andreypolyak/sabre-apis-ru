Branded Fares

Branded Fares

Оглавление:

  • Введение
  • Поиск (BargainFinderMaxRQ)
  • Расчет стоимости перелета по всем брендам (PO_AllBrandsPricingRQ)
  • Бронирование и расчет стоимости (EnhancedAirBookRQ, OTA_AirPriceLLSRQ)
    • Выбор бренда для всего бронирования
    • Посегментный выбор бренда
  • Получение информации о брендах перевозчика (GetMarketingTextRQ)
    • Расшифровка описания условий бренда

Введение

Branded Fares — инструмент Sabre, предназначенный для поиска, бронирования и оформления билетов по различным тарифным семействам или брендам, указанных перевозчиком. Для каждого бренда определяются условия, такие, как: наличие или отсутствие багажа, возможность обмена и возврата авиабилета, питание на борту, класс обслуживания и т.д. Тарифы разных брендов одного перевозчика могут иметь один класс бронирования, но различаться по коду тарифу.

На данный момент многие отечественные (включая SU, S7, U6 и UT) и иностранные перевозчики предлагают тарифные семейства, доступные с использованием Branded Fares.

Подробнее о Branded Fares можно прочитать на портале Agency eServices. Проверить поддержку Branded Fares определенным перевозчиком можно на портале Air Merchandising.

Поиск (BargainFinderMaxRQ)

По умолчанию в результатах поиска сервиса Bargain Finder Max предлагаются варианты перелетов, рассчитанные только по самым дешевым доступным тарифам. Это не всегда удобно для клиента, т.к. зачастую самый дешевый тариф не разрешает провоз багажа, возврат билета и другие дополнительные возможности.

Для получения расчетов стоимости по всем брендам одновременно для всех найденных перелетов необходимо указать значение true у атрибута /OTA_AirLowFareSearchRQ/TravelerInfoSummary/PriceRequestInformation/TPA_Extensions/BrandedFareIndicators@MultipleBrandedFares в запросе к сервису BargainFinderMaxRQ.

Также в запросе можно указать, что необходимо вернуть в поисковой выдаче код бренда только для самого дешевого варианта расчета. Для этого необходимо указать значение true у атрибута /OTA_AirLowFareSearchRQ/TravelerInfoSummary/PriceRequestInformation/TPA_Extensions/BrandedFareIndicators@SingleBrandedFare в запросе к сервису BargainFinderMaxRQ.

Расчет стоимости перелета по всем брендам (PO_AllBrandsPricingRQ)

Пожалуйста, сообщите своему куратору в Sabre, если вы захотите использовать использовать этот сервис. В противном случае, доступ к сервису может быть ограничен.

Для получения расчета стоимости выбранного перелета по всем доступным брендам используется сервис PO_AllBrandsPricingRQ.

Сервис не требует наличия созданного бронирования, а вся информация о перелете должна быть указана в запросе:

  • /AllBrandsPricingRQ/AirItinerary/OriginDestinationOptions/OriginDestinationOption/FlightSegment — сегменты:
    • /@SegmentNumber — номер сегмента в бронировании
    • /@SegmentType — тип сегмента (авиасегмент — A, наземный сегмент (ARNK) — K)
    • /@FlightNumber — номер рейса
    • /@DepartureDate — дата вылета
    • /@ResBookDesigCode — класс бронирования
    • /DepartureAirport/@LocationCode — аэропорт отправления
    • /ArrivalAirport/@LocationCode — аэропорт прибытия
    • /MarketingAirline/@Code — код маркетингового перевозчика
    • /Marriage/@Status — статус соединенного сегмента (первый сегмент в цепочке соединенных сегментов — S, средний сегмент в цепочке соединенных сегментов — P, последний сегмент в цепочке соединенных сегментов — E)
  • /AllBrandsPricingRQ/PriceRequestInformation/PassengerTypes/PassengerType — пассажиры:
    • /@Code — категория пассажиров
    • /@Count — количество пассажиров данной категории

Бронирование и расчет стоимости (EnhancedAirBookRQ, OTA_AirPriceLLSRQ)

Алгоритм внесения информации о бренде различается в зависимости от того применяется ли один и тот же бренд для всех сегментов бронирования, применяется несколько разных брендов для разных сегментов или же для некоторых сегментов бренды применяются, а для некоторых нет.

Для получения этой информации необходимо проанализировать ответ сервиса BargainFinderMaxRQ:

  • если все элементы /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/PassengerFare/TPA_Extensions/FareComponents/FareComponent (для самой дешевой комбинации брендов) или /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/TPA_Extensions/AdditionalFares/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/PassengerFare/TPA_Extensions/FareComponents/FareComponent (для дополнительных вариантов) содержат один и тот же код бренда в качестве значения атрибута /@BrandID, то для всего бронирования должен быть выбран один бренд
  • если все указанные выше элементы имеют различный атрибут /@BrandID или некоторые из указанных элементов не имеют атрибута /@BrandID, то при бронировании должен применяться посегментный выбор бренда

Или ответ сервиса PO_AllBrandsPricingRQ:

  • если все элементы /AllBrandsPricingRS/Summary/PassengerDetails/PassengerDetail/PassengerFare/FareCalculation/FareComponents/FareComponent содержат один и тот же код бренда в качестве значения атрибута /@BrandID, то для всего бронирования должен быть выбран один бренд
  • если все указанные выше элементы имеют различный атрибут /@BrandID или некоторые из указанных элементов не имеют атрибута /@BrandID, то при бронировании должен применяться посегментный выбор бренда

При создании бронирования при помощи сервиса EnhancedAirBookRQ, а также расчете стоимости бронирования при помощи сервиса OTA_AirPriceLLSRQ необходимо указать код бренда в (Brand ID) в элементе /EnhancedAirBookRQ/OTA_AirPriceRQ/PriceRequestInformation/OptionalQualifiers/PricingQualifiers/Brand или /OTA_AirPriceRQ/PriceRequestInformation/OptionalQualifiers/PricingQualifiers/Brand соответственно.

См. также Создание бронирований.

Посегментный выбор бренда

Обратите внимание на то, что многие перевозчики имеют жесткие требования относительно комбинации различных брендов. Не рекомендуется комбинировать их самостоятельно! Результаты поиска Bargain Finder Max будут содержать только те комбинации брендов, которые разрешены перевозчиком.

Для посегментного выбора бренда в случае получения расчетов по всем брендам в результатах поиска (сервис BargainFinderMaxRQ) необходимо определить соответствие между элементами /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/PassengerFare/TPA_Extensions/FareComponents/FareComponent (для самой дешевой комбинации брендов) или /OTA_AirLowFareSearchRS/PricedItineraries/PricedItinerary/TPA_Extensions/AdditionalFares/AirItineraryPricingInfo/PTC_FareBreakdowns/PTC_FareBreakdown/PassengerFare/TPA_Extensions/FareComponents/FareComponent (для дополнительных вариантов) и сегментами. Для этого каждый элемент /FareComponent содержит дочерние элементы /Segment, каждый из которых соответствует одному сегменту в маршруте перелета. Для установления точного соответствия между элементом /Segment и сегментами в путешествии используются атрибуты:

  • /Segment/@LegIndex — номер плеча
  • /Segment/@FlightIndex — номер сегмента в плече

При получении расчетов для всех брендов в ответе на запрос к сервису PO_AllBrandsPricingRQ каждый элемент /AllBrandsPricingRS/Summary/PassengerDetails/PassengerDetail/PassengerFare/FareCalculation/FareComponents/FareComponent будет содержать всю информацию о сегменте (или сегментах), к которому он относится, в дочернем элементе /AllBrandsPricingRS/Summary/PassengerDetails/PassengerDetail/PassengerFare/FareCalculation/FareComponents/FareComponent/FlightSegments.

После получения соответствия между брендами и сегментами необходимо в запросе к сервису создания бронирования EnhancedAirBookRQ или к сервису для расчета стоимости OTA_AirPriceLLSRQ указать:

  • последовательности сегментов, имеющих одинаковый бренд или не имеющих бренда в элементах (/EnhancedAirBookRQ)/OTA_AirPriceRQ/PriceRequestInformation/OptionalQualifiers/PricingQualifiers/ItineraryOptions/SegmentSelect со следующими атрибутами:
    • /@Number — номер сегмента начала последовательности
    • /@EndNumber — номер сегмента конца последовательности (опционально)
    • /@RPH — порядковый номер последовательности
  • коды брендов в последовательно расположенных элементах (/EnhancedAirBookRQ)/OTA_AirPriceRQ/PriceRequestInformation/OptionalQualifiers/PricingQualifiers/Brand, где в качестве значения атрибута /@RPH должен быть номер последовательности из предыдущего пункта. Обратите внимание на то, что если для каких-то сегментов код бренда не должен быть использован, то в качестве значения элемента /Brand для этой последовательности должна быть пустая строка

Получение информации о брендах перевозчика (GetMarketingTextRQ)

Для получения общей информации по всем доступным брендам перевозчика используется сервис GetMarketingTextRQ. Расшифровка описания условий бренда

Ответ на запрос к сервису GetMarketingTextRQ содержит описание бренда, располагающееся в текстовом виде в элементе /GetMarketingTextRS/BrandsMarketingTextResponse/CarrierBrandsData/BrandProgram/BrandsData/Brand/MarketingText/Text в структурированном виде как перечень различных услуг.

Расшифровка описания бренда производится по следующим правилам:

  • каждая услуга в описании бренда начинается с символов //
  • услуга имеет следующий формат описания: //субкод услуги/код применимости услуги/текстовое описание услуги
  • справочник субкодов услуг составляется ATPCO и доступен на сайте организации
  • возможные варианты кодов применимости услуги:
    • F — услуга предоставляется бесплатно
    • N — услуга не предоставляется
    • C — услуга предоставляется платно
    • D — услуга отображается, но не предоставляется

Пример:

//04Y/F/CHANGE BEFORE DEPARTURE//04Z/F/CHANGE
AFTER DEPARTURE//040/F/REFUND BEFORE
DEPARTURE//041/F/REFUND AFTER
DEPARTURE//PSA/F/SEAT RESERVATION//0AW/F/NON
ALCOHOLIC DRINK//0L5/F/CARRY ON
HAND BAG//0CC/F/CHECKED BAG FIRST//03P/F/PRIORITY
CHECK IN//0G6/F/PRIORITY BOARDING//0BX/F/LOUNGE
ACCESS

Расшифровка:

  • бесплатный обмен билета до вылета
  • бесплатный обмен билета после вылета
  • бесплатный возврат билета до вылета
  • бесплатный возврат билета после вылета
  • бесплатный выбор места в салоне
  • бесплатные безалкогольные напитки на борту
  • бесплатная ручная кладь
  • бесплатное место багажа
  • бесплатная приоритетная регистрация на рейс
  • бесплатная приоритетная посадка в самолет
  • бесплатный доступ в зал ожидания

Пример:
