Алгоритм бронирования мест в салоне

Схема

Чтение бронирования (TravelItineraryReadRQ)

Для чтения бронирования используется сервис TravelItineraryReadRQ.

В качестве значения атрибута /TravelItineraryReadRQ/UniqueID/@ID в запросе указывается код бронирования (PNR Record Locator).

В результате чтения бронирования должен быть составлен список сегментов, для которых необходимо получить карту мест.

Пример запроса
Копировать 
<TravelItineraryReadRQ Version="3.9.0" xmlns="http://services.sabre.com/res/tir/v3_9">
  <MessagingDetails>
    <SubjectAreas>
      <SubjectArea>FULL</SubjectArea>
    </SubjectAreas>
  </MessagingDetails>
  <UniqueID ID="SVDMUS"/>
</TravelItineraryReadRQ>

Получение карты мест (EnhancedSeatMapRQ)

Для получения карты мест используется сервис EnhancedSeatMapRQ.

В запросе необходимо указать:

  • /EnhancedSeatMapRQ/SeatMapQueryEnhanced/RequestType — тип запроса, всегда Payload
  • /EnhancedSeatMapRQ/SeatMapQueryEnhanced/Flight/@origin и /@destination — код аэропортов вылета и прилета
  • /EnhancedSeatMapRQ/SeatMapQueryEnhanced/Flight/DepartureDate — дата вылета
  • /EnhancedSeatMapRQ/SeatMapQueryEnhanced/Flight/Marketing — номер рейса
  • /EnhancedSeatMapRQ/SeatMapQueryEnhanced/Flight/Marketing/@carrier — код маркетингового перевозчика
  • /EnhancedSeatMapRQ/SeatMapQueryEnhanced/CabinDefinition/RBD — класс бронирования
  • /EnhancedSeatMapRQ/SeatMapQueryEnhanced/FareAvailQualifiers/@fareBasisCode — код тарифа, к которому относится текущий сегмент
  • /EnhancedSeatMapRQ/SeatMapQueryEnhanced/FareAvailQualifiers/@passengerType — код категории пассажира
  • /EnhancedSeatMapRQ/SeatMapQueryEnhanced/FareAvailQualifiers/@accompaniedByInfantInd — признак путешествия с младенцем без места
  • /EnhancedSeatMapRQ/SeatMapQueryEnhanced/FareAvailQualifiers/TravellerID — номер пассажира в бронировании (1, 2, 3 и т.д.)
  • /EnhancedSeatMapRQ/SeatMapQueryEnhanced/POS/PCC — текущий PCC, в котором будет производиться бронирование мест

Обратите внимание на то, что для получения карты мест для нескольких сегментов и (или) для нескольких категорий пассажиров требуется отправить запрос для получения карты мест для каждого сочетания рейса и категории пассажира.

Бронирование мест в салоне (PassengerDetailsRQ)

Для бронирования мест в салоне используется сервис-обертка PassengerDetailsRQ (см. также Создание бронирований).

Сервис PassengerDetailsRQ выполнит следующие операции:

  1. Игнорирование бронирования в текущей сессии
  2. Чтение указанного бронирования
  3. Бронирование мест в салоне
  4. Сохранение изменений
  5. Чтение измененного бронирования
  6. Игнорирование открытого бронирования

В случае возникновения ошибок выполнение запроса будет прервано, а бронирование игнорировано.

В запросе необходимо указать:

  • /PassengerDetailsRQ/@HaltOnError — прекращение выполнения запроса в случае возникновения ошибки (значение true)
  • /PassengerDetailsRQ/@IgnoreOnError — игнорирование бронирования в случае возникновения ошибки (значение true)
  • /PassengerDetailsRQ/PostProcessing/@IgnoreAfter — игнорирование бронирования после выполнения запроса
  • /PassengerDetailsRQ/PostProcessing/@RedisplayReservation — чтение бронирования после выполнения запроса. Обратите внимание на то, что после успешного сохранения бронирования оно будет сначала прочтено, а потом игнорировано
  • /PassengerDetailsRQ/PostProcessing/EndTransactionRQ/EndTransaction/@Ind — сохранение бронирования после выполнения бронирования
  • /PassengerDetailsRQ/PostProcessing/EndTransactionRQ/Source/@ReceivedFrom — значение поля Received From при сохранении бронирования
  • /PassengerDetailsRQ/PreProcessing/@IgnoreBefore — игнорирование содержимого сессии до момента выполнения запроса (значение true)
  • /PassengerDetailsRQ/PreProcessing/UniqueID/@ID — код бронирования (PNR Record Locator)
  • /PassengerDetailsRQ/SpecialReqDetails/AirSeatRQ/Seats/Seat — место в салоне (отдельный элемент для каждого места в салоне) /@NameNumber — номер пассажира /@Number — номер места /@SegmentNumber — номер сегмента в бронировании

Оформление EMD для платных мест

Для платных мест обязательно требуется оформление EMD. Обратите внимание на то, что EMD должны быть оформлены в бронировании после билетов.

Процесс оформления EMD подробно описан в разделе Оформление билетов и EMD.

Удаление забронированного места (AirSeatCancelLLSRQ)

Для удаления забронированного места используется сервис AirSeatCancelLLSRQ.

В запросе необходимо указать:

  • для удаления всех забронированных мест:
    • /AirSeatCancelRQ/Seats/Seat/@All — значение true
  • для удаления всех забронированных мест на рейсах определенного перевозчика:
    • /AirSeatCancelRQ/Seats/Seat/VendorPrefs/Airline/@Code — код перевозчика
  • для удаления всех забронированных мест на определенном сегменте:
    • /AirSeatCancelRQ/Seats/Seat/@SegmentNumber — номер сегмента в бронировании
  • для удаления определенного забронированного места на определенном сегменте:
    • /AirSeatCancelRQ/Seats/Seat/@SegmentNumber — номер сегмента в бронировании
    • /AirSeatCancelRQ/Seats/Seat/@SeatNumber — номер места
  • для удаления определенного забронированного места для определенного пассажира:
    • /AirSeatCancelRQ/Seats/Seat/@SegmentNumber — номер сегмента в бронировании
    • /AirSeatCancelRQ/Seats/Seat/@NameNumber — номер пассажира в бронировании
