Переход в другой PCC (ContextChangeLLSRQ)

Для выписки билета или EMD в другом PCC необходимо перейти в него, отправив запрос к сервису ContextChangeLLSRQ. Запрос не потребуется, если билет или EMD нужно оформить в iPCC.

Обратите внимание на то, что для перехода в другой PCC требуется наличие Branch Access между ним и iPCC, в котором была создана сессия. Подробнее о Branch Access см. в разделе Конфигурация Sabre.

В качестве значения атрибута /ContextChangeRQ/ChangeAAA/@PseudoCityCode в запросе указывается PCC, в котором должен быть выпущен билет или EMD.

Пример запроса
Копировать 
<ContextChangeRQ ReturnHostCommand="true" Version="2.0.3" xmlns="http://webservices.sabre.com/sabreXML/2011/10">
  <ChangeAAA PseudoCityCode="8XFG"/>
</ContextChangeRQ>

Пример ответа
Копировать

  <stl:ApplicationResults status="Complete">
    <stl:Success timeStamp="2016-12-10T04:32:47-06:00">

Чтение бронирования (TravelItineraryReadRQ)

Для чтения бронирования используется сервис TravelItineraryReadRQ.

В качестве значения атрибута /TravelItineraryReadRQ/UniqueID/@ID в запросе указывается код бронирования (PNR Record Locator).

Пример запроса
Копировать 
<TravelItineraryReadRQ Version="3.8.0" xmlns="http://services.sabre.com/res/tir/v3_8">
  <MessagingDetails>
    <SubjectAreas>
      <SubjectArea>FULL</SubjectArea>
    </SubjectAreas>
  </MessagingDetails>
  <UniqueID ID="DJDVWE"/>
</TravelItineraryReadRQ>

Оформление билета или EMD (AirTicketLLSRQ)

Для оформления билета или EMD используется сервис AirTicketLLSRQ.

Существует несколько стратегий оформления билетов:

  1. оформление всех билетов для всех пассажиров по всем PQ одним запросом
  2. оформление всех билетов для каждого PQ отдельным запросом
  3. оформление билета для каждого пассажира отдельным запросом

Например, если в бронировании есть 3 PQ для 4 пассажиров: 1 PQ для 2 взрослых пассажиров (ADT), 1 PQ для 1 ребенка (CNN) и 1 PQ для 1 младенца (INF), то в зависимости от выбранной стратегии должно быть отправлено:

  1. 1 запрос к AirTicketLLSRQ для всех PQ
  2. 3 запроса к AirTicketLLSRQ для каждого PQ
  3. 4 запроса к AirTicketLLSRQ для каждого пассажира

Обратите внимание на то, что в случае использования третьей стратегии, при наличии в бронировании нескольких пассажиров одной категории и при включенной настройке Price Retention Ticketing Alert (см. Настройки PCC при попытке оформить билет будет вызвано предупреждение). Для решения этой проблемы необходимо выполнить одно из действий:

  • отключить настройку Price Retention Ticketing Alert
  • использовать другую стратегию оформления билетов
  • добавить обработку предупреждений (см. Настройки PCC)

В данном документе в примерах используется вторая стратегия, однако указанные здесь рекомендации применимы к любой из них.

При оформлении билета или билетов необходимо указать:

  • одну или несеколько форм оплаты (см. примеры ниже)
  • величину комиссии в процентах (/AirTicketRQ/OptionalQualifiers/MiscQualifiers/Commission/@Percent) или абсолютных значениях (/@Amount)
  • один или несколько номеров PQ, по которым будет производиться оформление билетов (/AirTicketRQ/OptionalQualifiers/PricingQualifiers/PriceQuote/Record/@Number для одного PQ, если PQ несколько, то в атрибуте /@Number задается номер первого PQ, а в атрибуте /@EndNumber — номер последнего PQ)

Дополнительно в запросе можно указать:

  • номер пассажира (/AirTicketRQ/OptionalQualifiers/PricingQualifiers/PriceQuote/NameSelect/@NameNumber)
  • текст для добавления в полe Endorsement билета (/AirTicketRQ/OptionalQualifiers/MiscQualifiers/Endorsement/Text). Для переопределения значения этого поля необходимо дополнительно указать значение true у атрибута /AirTicketRQ/OptionalQualifiers/MiscQualifiers/Endorsement/@Override
  • добавление даты рождения пассажира из SSR в поле Endorsement билета (/AirTicketRQ/OptionalQualifiers/MiscQualifiers/DateOfBirth)

Также необходимо указать 10 в качестве значения атрибута /AirTicketRQ/@NumResponses.

Чтение бронирования (TravelItineraryReadRQ)

Обратите внимание на то, что данный рекомендованный процесс оформления билетов и EMD разработан с учетом того, что в PCC, в котором производится оформление билетов включена настройка Automatically End Transaction at Ticketing (см. Настройки PCC).

Если эта настройка не включена, то перед чтением бронирования его необходимо сохранить при помощи запроса к сервису EndTransactionLLSRQ (см. Редактирование бронирований).
