Рекомендации по работе с Sabre APIs

Конфигурация Sabre

Оглавление:

  • Глоссарий
  • Обязательные настройки PCC (TJR)
  • Необязательные настройки PCC (TJR)
  • Настройки учетной записи (EPR)
  • Приватные тарифы

Глоссарий

PCC (Pseudo City Code) — уникальный идентификатор точки продаж (офиса) в системе Sabre. Именно по этому идентификатору Sabre и перевозчики идентифицируют агентства.

Одно агентство может иметь несколько PCC, например, для разных точек продаж или разных целей (например, для бронирования и оформления билетов).

При работе через Sabre APIs используется специальный вид PCC — iPCC (Internet PCC). В iPCC подключен инструмента для поиска перелетов с минимальной ценой — Bargain Finder Max (BFM). Также iPCC используется для создания сессий для работы с сервисами Sabre. Во всех остальных аспектах iPCC ничем не отличается от любого другого PCC.

При работе с Sabre можно переключаться или переходить из одного PCC в другой, для этого предусмотрен механизм открытия доступа — Branch Access.

Разные PCC могут быть использованы для следующих целей:

  • поиск вариантов перелетов (всегда используется iPCC)
  • создание бронирования
  • оформление авиабилета

В рекомендуемом рабочем процессе в данном Руководстве рассмотрены следующие варианты:

  • поиск и бронирование осуществляются в iPCC, оформление билета в другом (или других) PCC
  • поиск, бронирование и оформление билетов осуществляются в iPCC

У каждого PCC существует ряд настроек, которые называются TJR (Travel Journal Record). Настройки TJR определяют различные аспекты работы системы Sabre и применимы, как для работы через Sabre APIs, так и для работы через терминал. Указанные ниже обязательные и необязательные настройки рекомендуется устанавливать во всех используемых PCC (включая iPCC).

PCC могут содержать учетные записи пользователей, которые называются EPR (Employee Profile Record). Каждая учетная запись состоит из следующих параметров:

  • референция (Agent ID, Username) — содержит от 1 до 6 цифр и используется для авторизации в системе
  • подпись агента (Agent Sign) — содержит 3 символа первый из которых "A", используется в качестве идентификатора агента в истории операций
  • пароль (Passcode, Password) — содержит 7 или 8 символов
  • ключевые слова (Keywords) — каждое ключевое слово содержит 6 букв, используются для подключения определенных функций в системе для данной учетной записи

Настройки PCC и EPR могут быть изменены супервайзером агентства после прохождения соответствующего тренинга или по заявке, направленной в Sabre Helpdesk.

Обязательные настройки PCC (TJR)

Данное руководство составлено с учетом того, что во всех PCC включены следующие настройки:

  • Store Passenger Type in PNR — хранение типа пассажира в PNR
  • Enhanced PQ — использование Enhanced PQ
  • Automatically End Transaction at Ticketing — автоматическое сохранение бронирования после оформления билета
  • Auto Segment Arrangement — автоматическая расстановка сегментов
  • PSPT Warning Message — предупреждение об отсутствии паспорта при попытке оформить билет

Необязательные настройки PCC (TJR)

  • Two Forms of Payment — использование двух форм оплаты в одном билете
  • Price Retention Ticketing Alert — предупреждение об изменении стоимости при попытке оформить билет
  • Ticket From Stored Fare — оформление билета по сохраненному расчету без перерасчета стоимости
  • Stored Fare Warning at Ticketing — предупреждение об изменении стоимости билета при оформлении

Настройки учетной записи (EPR)

Обязательные ключевые слова для учетной записи, используемой для работы с Sabre APIs:

  • SUBAAA — возможность перейти в другой PCC
  • BSRDSP — возможность получить курсы валют

Приватные тарифы

При работе с приватными тарифами необходимо убедиться, что они будут зафайлированы в тех PCC, в которых производится поиск, расчет стоимости и оформление билетов.
